## 1. 模型导出格式

### 1.1 格式总览

| 格式 | 后缀 | 推理框架 | 硬件支持 | 量化支持 | 典型延迟 (YOLOv8s, 640) |
|------|------|---------|---------|---------|----------------------|
| PyTorch | .pt | PyTorch | CPU/GPU | 有限 | ~15ms (RTX 4060) |
| TorchScript | .torchscript | LibTorch | CPU/GPU | 不支持 | ~12ms |
| ONNX | .onnx | ONNX Runtime | CPU/GPU/NPU | FP16/INT8 | ~8ms |
| TensorRT | .engine | TensorRT | NVIDIA GPU | FP16/INT8 | ~3ms |
| OpenVINO | _openvino_model | OpenVINO | Intel CPU/GPU/VPU | INT8 | ~6ms (CPU) |
| CoreML | .mlpackage | CoreML | Apple Silicon | FP16 | ~5ms (M2) |
| TFLite | .tflite | TensorFlow Lite | CPU/GPU/NPU | INT8 | ~10ms (CPU) |
| NCNN | .param/.bin | NCNN | ARM CPU | FP16/INT8 | ~8ms (手机) |

### 1.2 各格式详解

**ONNX（Open Neural Network Exchange）**

```
通用中间格式, 几乎所有推理框架都支持
优点: 通用性强, 工具链成熟 (onnxsim, onnx-graphsurgeon)
缺点: 算子兼容性问题 (某些自定义算子需要注册)
```

**TensorRT**

```
NVIDIA 专属, GPU 上性能最强
优点: 层融合, kernel 自动调优, 最大化 GPU 利用率
缺点: 仅 NVIDIA 平台, 版本绑定 CUDA 版本
```

**OpenVINO**

```
Intel 专属, CPU 上性能最强
优点: Intel CPU/GPU/VPU 全平台, INT8 量化成熟
缺点: 仅 Intel 平台
```

**CoreML / TFLite**

```
移动端部署
CoreML: Apple 设备, 硬件加速 (Neural Engine)
TFLite: Android 设备, GPU delegate / NNAPI
```

## 2. 量化（Quantization）

### 2.1 量化基础

```
FP32 (32位浮点)    精度最高, 体积最大, 速度最慢
    |
FP16 (16位半精度)   精度略降, 体积减半, 速度提升 1.5~2x
    |
INT8 (8位整数)      精度损失可控, 体积 1/4, 速度提升 2~4x
    |
INT4 (4位整数)      精度损失较大, 体积 1/8, 速度提升 3~5x
```

### 2.2 量化策略

**PTQ（Post-Training Quantization，训练后量化）**

```
流程:
  训练好的 FP32 模型
       |
  [校准] 使用少量数据 (100~1000 张) 统计激活值范围
       |
  [量化] 将权重和激活从 FP32 映射到 INT8
       |
  INT8 模型

优点: 无需重新训练, 简单快速
缺点: 精度损失可能较大 (尤其是 INT8)
```

**QAT（Quantization-Aware Training，量化感知训练）**

```
流程:
  FP32 模型
       |
  [插入伪量化节点] 模拟量化误差
       |
  [微调训练] 让模型适应量化误差 (通常 5~10 epochs)
       |
  [真正量化] 导出 INT8 模型

优点: 精度损失小
缺点: 需要训练数据和额外训练时间
```

### 2.3 量化精度对比

以 YOLOv8s 在 COCO val2017 上为例：

| 量化方式 | 精度 | mAP@0.5:0.95 | 模型大小 | 推理速度 (RTX 4060) |
|---------|------|-------------|---------|-------------------|
| FP32 | 原始 | 44.9% | 22.5 MB | 15ms |
| FP16 | 半精度 | 44.8% | 11.3 MB | 8ms |
| INT8 PTQ | 量化后 | 43.2% | 5.9 MB | 4ms |
| INT8 QAT | 量化感知 | 44.1% | 5.9 MB | 4ms |

> 注：数值为示意性参考，实际因硬件和配置而异。

## 3. 模型剪枝（Pruning）

### 3.1 非结构化剪枝

```
将单个权重置为 0 (稀疏化)
优点: 灵活, 精度损失小
缺点: 需要专用硬件/库支持稀疏计算
      普通 GPU 上加速有限
```

### 3.2 结构化剪枝

```
删除整个通道/层
优点: 无需特殊硬件支持
      直接减少计算量和模型大小
缺点: 精度损失可能较大
      需要 fine-tuning 恢复精度

流程:
  训练好的模型
       |
  [通道重要性评估] L1范数 / BatchNorm gamma / Taylor展开
       |
  [剪枝] 删除不重要的通道
       |
  [Fine-tune] 恢复精度 (通常 10~20 epochs)
       |
  精简模型
```

### 3.3 YOLO 剪枝示例

```python
# 使用 YOLOv8 的剪枝功能 (Ultralytics)
from ultralytics import YOLO

model = YOLO("yolov8s.pt")
model.train(data="coco.yaml", epochs=100)  # 先正常训练

# 剪枝 (Ultralytics 内置)
model.prune(amount=0.3)  # 剪掉 30% 的通道
model.train(data="coco.yaml", epochs=20)   # fine-tune 恢复
```

## 4. 知识蒸馏（Knowledge Distillation）

### 4.1 核心思想

用大模型（Teacher）指导小模型（Student）学习。

```
Teacher (YOLOv8l)          Student (YOLOv8n)
  |                           |
  |  特征图/输出               |  特征图/输出
  |                           |
  +------> 蒸馏损失 <---------+
           L_distill

总损失 = L_task + alpha * L_distill

L_task:    原始检测损失
L_distill: 蒸馏损失 (KL散度 / MSE / 特征对齐)
alpha:     蒸馏权重
```

### 4.2 蒸馏策略

| 策略 | 蒸馏对象 | 适用场景 |
|------|---------|---------|
| Logit Distillation | 最终输出 logits | 通用 |
| Feature Distillation | 中间特征图 | 结构相似的模型对 |
| Relation Distillation | 样本间关系 | Teacher-Student 结构差异大 |

## 5. Ultralytics 一键导出

### 5.1 命令行导出

```bash
# 导出 ONNX
yolo export model=yolov8s.pt format=onnx simplify=True

# 导出 TensorRT (FP16)
yolo export model=yolov8s.pt format=engine half=True

# 导出 TensorRT (INT8)
yolo export model=yolov8s.pt format=engine int8=True data=coco.yaml

# 导出 OpenVINO
yolo export model=yolov8s.pt format=openvino

# 导出 CoreML
yolo export model=yolov8s.pt format=coreml

# 导出 TFLite
yolo export model=yolov8s.pt format=tflite
```

### 5.2 Python API 导出

```python
from ultralytics import YOLO

model = YOLO("yolov8s.pt")

# 导出 ONNX
model.export(format="onnx", simplify=True, opset=12)

# 导出 TensorRT FP16
model.export(format="engine", half=True)

# 导出 TensorRT INT8
model.export(format="engine", int8=True, data="coco.yaml")
```

### 5.3 导出参数说明

| 参数 | 说明 | 默认值 |
|------|------|-------|
| `format` | 导出格式 | onnx |
| `half` | FP16 半精度 | False |
| `int8` | INT8 量化 | False |
| `simplify` | ONNX 简化 | True |
| `opset` | ONNX opset 版本 | 17 |
| `dynamic` | 动态 batch size | False |
| `nms` | 内置 NMS | False |
| `data` | INT8 校准数据 | None |

## 6. 性能对比总结

以 YOLOv8s, COCO, 640x640, RTX 4060 Laptop 为例：

| 部署方案 | mAP@0.5:0.95 | 延迟 (ms) | 吞吐量 (FPS) | 模型大小 | 内存占用 |
|---------|-------------|----------|-------------|---------|---------|
| PyTorch FP32 | 44.9% | 15 | 67 | 22.5 MB | 512 MB |
| ONNX FP32 | 44.9% | 8 | 125 | 22.5 MB | 256 MB |
| ONNX FP16 | 44.8% | 5 | 200 | 11.3 MB | 128 MB |
| TensorRT FP16 | 44.8% | 3 | 333 | 11.3 MB | 96 MB |
| TensorRT INT8 | 43.2% | 2 | 500 | 5.9 MB | 64 MB |
| OpenVINO CPU | 44.9% | 12 | 83 | 22.5 MB | 384 MB |

> 注：数值为示意性参考，实际因硬件驱动版本、batch size 等因素而异。
