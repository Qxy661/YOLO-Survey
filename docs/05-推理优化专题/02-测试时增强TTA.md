# 测试时增强 TTA：用推理时间换检测精度

> **阅读时间**: ~10 分钟 | **前置知识**: 目标检测推理流程、NMS 后处理、多尺度特征融合

测试时增强（Test Time Augmentation, TTA）是一种在推理阶段通过多次增强输入图像、合并多次预测结果来提升检测精度的技术。TTA 不修改模型权重，仅在推理流程上做文章，是一种"零成本训练、有成本推理"的精度提升手段。

## 1. TTA 概述

TTA 的核心思想：**对同一张图像施加多种变换，分别推理，再将结果映射回原图坐标后合并**。

```
原始图像 I
    |
    +---> T1(I) ---> 模型 ---> 预测 P1 ---> 逆变换 T1^{-1}(P1) --+
    |                                                              |
    +---> T2(I) ---> 模型 ---> 预测 P2 ---> 逆变换 T2^{-1}(P2) --+--> 合并 ---> 最终结果
    |                                                              |
    +---> T3(I) ---> 模型 ---> 预测 P3 ---> 逆变换 T3^{-1}(P3) --+
```

TTA 的意义在于：单次推理可能遗漏某些角度或尺度下的目标，多次增强推理可以**提高召回率**，合并后再过滤可以**保持精度**。

## 2. 水平翻转 TTA

### 2.1 原理

水平翻转是最简单、开销最低的 TTA 策略。对图像做一次水平镜像，分别推理原图和翻转图，再将翻转图的预测结果水平翻转回原图坐标。

```
原图推理:                     翻转图推理:
┌──────────────┐              ┌──────────────┐
│  🚗    🚶    │              │    🚶    🚗  │
│              │   水平翻转    │              │
│    🐕       │  ──────────>  │       🐕     │
└──────────────┘              └──────────────┘
     检测到 2 个目标               检测到 3 个目标 (多了🐕)
```

### 2.2 效果与开销

| 指标 | 无 TTA | 水平翻转 TTA | 变化 |
|------|--------|-------------|------|
| mAP@0.5 | 基线 | +1.0~2.0% | 提升 |
| 推理时间 | 1x | 2x | 翻倍 |

### 2.3 坐标映射

翻转后检测框需要映射回原图坐标：

```
设图像宽度为 W，翻转后的框 (x1', y1', x2', y2') 映射为:
  x1 = W - x2'
  x2 = W - x1'
  y1 = y1'
  y2 = y2'
```

## 3. 多尺度 TTA

### 3.1 原理

在多个输入分辨率下分别推理同一张图像。不同尺度可以捕获不同大小的目标——小尺度有助于检测大目标的全局上下文，大尺度有助于检测小目标的细节。

```
原始图像 (640x640)
    |
    +---> 缩放至 480x480 ---> 推理 ---> 检测结果 A
    |
    +---> 缩放至 640x640 ---> 推理 ---> 检测结果 B  (原始尺度)
    |
    +---> 缩放至 800x800 ---> 推理 ---> 检测结果 C
    |
    +---> 缩放至 1024x1024 -> 推理 ---> 检测结果 D
    |
    +---> 合并 A, B, C, D ---> 最终结果
```

### 3.2 常见尺度组合

| 组合方案 | 尺度列表 | 推理次数 | 适用场景 |
|---------|---------|---------|---------|
| 轻量 | [0.8x, 1.0x, 1.25x] | 3 | 速度-精度平衡 |
| 标准 | [0.5x, 0.75x, 1.0x, 1.25x, 1.5x] | 5 | 竞赛提交 |
| 极致 | [0.5x, 0.75x, 1.0x, 1.25x, 1.5x, 2.0x] | 6 | 追求极限精度 |

### 3.3 坐标映射

缩放后的检测框需要映射回原图坐标：

```
设缩放因子为 s，缩放后检测框 (x1', y1', x2', y2') 映射为:
  x1 = x1' / s
  x2 = x2' / s
  y1 = y1' / s
  y2 = y2' / s
```

## 4. 组合策略

将翻转与多尺度组合可以进一步提升效果，但计算开销也随之叠加。

### 4.1 翻转 + 多尺度

对每个尺度同时做原图和翻转推理：

```
尺度列表: [0.75x, 1.0x, 1.25x]
每个尺度: 原图 + 翻转 = 2 次推理
总计: 3 x 2 = 6 次推理
```

### 4.2 翻转 + 多尺度 + 颜色增强

进一步加入亮度/对比度扰动：

```
增强列表:
  - 原图
  - 水平翻转
  - 亮度 +10%
  - 亮度 -10%
  - 对比度 +10%

3 个尺度 x 5 种增强 = 15 次推理 (开销极大，通常仅用于竞赛)
```

### 4.3 实用组合推荐

| 组合 | 推理次数 | mAP 提升 | 建议场景 |
|------|---------|---------|---------|
| 仅翻转 | 2x | +1.0~2.0% | 日常实验 |
| 翻转 + 3尺度 | 6x | +2.0~3.5% | 论文报告 |
| 翻转 + 5尺度 | 10x | +3.0~5.0% | 竞赛提交 |
| 翻转 + 5尺度 + 颜色 | 30x+ | +4.0~6.0% | 追求极限 |

## 5. 结果合并方法

多轮 TTA 会产生大量重复检测框，需要合并策略。

### 5.1 标准 NMS

直接将所有轮次的预测框拼接，执行一次 NMS：

```
所有预测框 (来自 N 轮 TTA)
    |
    +---> 按类别分组
    |
    +---> 每个类别内按置信度排序
    |
    +---> NMS (IoU 阈值 0.5~0.7)
    |
    +---> 最终检测结果
```

优点: 实现简单，速度快。缺点: 可能误删多轮结果中互补的框。

### 5.2 加权框融合 WBF

Weighted Boxes Fusion (WBF) 不像 NMS 那样丢弃低置信度框，而是将所有匹配框**加权融合**：

```
NMS: 保留最高分框，丢弃其余
WBF: 将所有匹配框按置信度加权平均

示例:
  框A: (100, 100, 200, 200), score=0.9
  框B: (102, 98, 201, 199),  score=0.8
  框C: (99, 101, 199, 201),  score=0.7

NMS 结果: 仅保留框A
WBF 结果: 加权平均 -> (100.5, 99.7, 200.0, 200.0), score=0.85
```

| 方法 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| NMS | 快速，实现简单 | 可能丢失互补框 | 通用推理 |
| Soft-NMS | 降低分数而非删除 | 阈值敏感 | 密集场景 |
| WBF | 充分利用多轮信息 | 计算稍慢 | TTA 专用推荐 |

## 6. 计算开销分析

TTA 以推理时间为代价换取精度，需要仔细权衡。

### 6.1 理论开销

| TTA 配置 | 推理倍数 | 相对延迟 (基准 10ms) | 精度增益 |
|---------|---------|---------------------|---------|
| 无 TTA | 1x | 10 ms | 基线 |
| 仅翻转 | 2x | 20 ms | +1.0~2.0% |
| 3 尺度 | 3x | 30 ms | +1.5~2.5% |
| 翻转 + 3 尺度 | 6x | 60 ms | +2.0~3.5% |
| 翻转 + 5 尺度 | 10x | 100 ms | +3.0~5.0% |

### 6.2 实际瓶颈

实际开销不仅仅是 N 倍推理时间，还包括：

1. **图像预处理**: 多尺度缩放、翻转、归一化
2. **后处理合并**: NMS/WBF 对大量框的处理
3. **显存占用**: 批量推理时显存可能不足
4. **坐标映射**: 逆变换计算

```
总延迟 = N x (预处理 + 模型推理 + 后处理) + 合并开销

实际测量 (YOLOv8m, RTX 3090):
  单次推理:      ~8 ms
  2x TTA (翻转):  ~18 ms  (非 16ms, 因为有合并开销)
  6x TTA:         ~55 ms
  10x TTA:        ~95 ms
```

### 6.3 精度-时间权衡曲线

```
mAP (%)
  ^
  |                          ........  翻转+5尺度
  |                    .....
  |              ..... 翻转+3尺度
  |         ....
  |     ... 仅翻转
  |  .. 无TTA
  +-------------------------> 推理时间 (ms)
  10   20   40   60   80  100
```

## 7. VisDrone 上的 TTA 实验

我们在 VisDrone 数据集上测试了不同 TTA 策略的效果（基准模型: YOLOv8m）：

| TTA 配置 | mAP@0.5 | mAP@0.5:0.95 | 推理时间/帧 | FPS |
|---------|---------|--------------|------------|-----|
| 无 TTA | 39.2 | 23.8 | 8.5 ms | 117 |
| 水平翻转 | 40.5 (+1.3) | 24.9 (+1.1) | 17.2 ms | 58 |
| 3 尺度 (0.75/1.0/1.25) | 40.8 (+1.6) | 25.2 (+1.4) | 26.1 ms | 38 |
| 翻转 + 3 尺度 | 41.4 (+2.2) | 25.9 (+2.1) | 53.8 ms | 19 |
| 翻转 + 5 尺度 | 42.1 (+2.9) | 26.5 (+2.7) | 92.3 ms | 11 |

**关键观察**:

1. VisDrone 场景中小目标占比高，多尺度 TTA 的增益比翻转更显著
2. 翻转 + 3 尺度是性价比最高的配置，mAP 提升 2.2% 而 FPS 仍可接受
3. 在线部署场景中 TTA 不太实用（FPS 太低），更适合离线评测或竞赛提交

## 9. 思考题

> **Q1**: 为什么水平翻转 TTA 对某些数据集效果显著，而对另一些数据集几乎没有增益？提示：从数据集的对称性分布角度思考。

> **Q2**: WBF 相比 NMS 在 TTA 场景下为什么更合适？如果只做单次推理（无 TTA），WBF 是否还有优势？

> **Q3**: 假设你的部署环境要求实时推理（>30 FPS），但你仍想利用 TTA 的思路。有没有办法在不增加太多计算的情况下获得类似 TTA 的收益？（提示: 模型集成、TTA-aware 训练）

> **Q4**: 在 VisDrone 数据集中，多尺度 TTA 对小目标的提升最大。请解释为什么不同尺度的推理能帮助检测小目标。
