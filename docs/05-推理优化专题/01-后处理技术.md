# 后处理技术

> 后处理是目标检测推理流程的最后一步：将模型输出的大量候选框筛选为最终检测结果。
> NMS（Non-Maximum Suppression）是事实标准，但存在诸多变体和替代方案。
> 本专题梳理主流后处理技术及其在 YOLO 系列中的演进。

## 1. 标准 NMS 流程

### 1.1 算法步骤

```
输入: 候选框集合 B, 置信度 S, IoU 阈值 Nt
输出: 最终检测框集合 D

1. 按置信度降序排列 B
2. while B 非空:
   a. 选取置信度最高的框 b_i, 加入 D
   b. 计算 b_i 与 B 中剩余所有框的 IoU
   c. 删除 IoU > Nt 的框（即与 b_i 高度重叠的框）
   d. 从 B 中移除 b_i
3. 返回 D
```

### 1.2 流程图

```
原始候选框 (1000+)
       |
  [置信度过滤]  score > conf_thres (如 0.25)
       |
  过滤后候选框 (200+)
       |
  [按置信度降序排序]
       |
  [循环: 取最高分框]
       |
  [计算 IoU, 抑制重叠框]  IoU > iou_thres (如 0.45)
       |
  最终检测框 (N 个)
```

### 1.3 标准 NMS 的问题

- **一刀切：** IoU > 阈值就完全抑制，可能误删正确检测。
- **密集场景失效：** 密集排列的同类目标被相互抑制。
- **阈值敏感：** 阈值过低导致漏检，过高导致重复框。

## 2. Soft-NMS

> 来源：Bodla et al., "Soft-NMS", ICCV 2017

### 2.1 核心思想

不直接删除重叠框，而是根据 IoU **衰减置信度**。

```
标准 NMS:                  Soft-NMS:
框A(0.9) -> 保留           框A(0.9) -> 保留
框B(0.85, IoU=0.6) -> 删除 框B(0.85, IoU=0.6) -> 置信度降为 0.34
框C(0.7, IoU=0.3) -> 删除  框C(0.7, IoU=0.3) -> 置信度降为 0.58
```

### 2.2 两种衰减方式

**线性衰减：**
```
s_i = s_i * (1 - IoU(b_i, b_M))    if IoU > Nt
s_i = s_i                           otherwise
```

**高斯衰减（更平滑）：**
```
s_i = s_i * e^(-IoU(b_i, b_M)^2 / sigma)
```

### 2.3 优势

- 密集场景中保留更多正确检测。
- 不需要硬阈值做"删除/保留"二值决策。

## 3. DIoU-NMS

> 来源：Zheng et al., "Distance-IoU Loss", AAAI 2020

### 3.1 核心思想

用**中心点距离**替代或补充 IoU 作为抑制判据。

```
DIoU-NMS 判据:
  S_i = S_i                     if IoU - rho^2(b_i, b_M) / c^2 < Nt
  S_i = S_i * f(IoU)            otherwise

其中:
  rho = 两框中心点的欧氏距离
  c   = 最小封闭框的对角线长度
```

### 3.2 优势

```
场景: 两个相邻但不同的目标

标准 NMS:                    DIoU-NMS:
+---+ +---+                 +---+ +---+
| A | | B |                 | A | | B |
+---+ +---+                 +---+ +---+
  IoU > 阈值?                中心距离较大
  -> 删除 B                  -> 保留 B
  (错误)                     (正确)
```

- 即使 IoU 较高，只要中心距离远，也不会被抑制。
- 适合密集、遮挡场景。

## 4. WBF（Weighted Boxes Fusion）

> 来源：Solovyev et al., "Weighted Boxes Fusion", 2021

### 4.1 核心思想

不是"抑制"，而是**融合**重叠框的坐标和置信度。

```
标准 NMS:          Soft-NMS:          WBF:
从重叠框中          降低重叠框          融合重叠框的
选一个, 删除其余    的置信度            坐标和置信度

  +---+              +---+              +---+
  | A |              | A |              |   |
  |(0.9)             |(0.9)             |融合|
  +---+              +---+              |(0.88)
    +---+              +---+            +---+
    | B |              | B |
    |(0.8)|            |(0.24)|
    +---+              +---+
```

### 4.2 算法流程

```
1. 将所有框按置信度降序排列
2. 按 IoU 阈值聚类（IoU > 阈值的框归为同一簇）
3. 对每个簇:
   a. 加权平均坐标: x = sum(s_i * x_i) / sum(s_i)
   b. 置信度: s = sum(s_i) / N  (或取 max)
4. 输出融合后的框
```

### 4.3 适用场景

- **多模型集成：** 多个模型的检测结果融合。
- **TTA（Test-Time Augmentation）：** 不同增强版本的结果融合。
- **不适用于实时推理：** 需要所有候选框同时处理。

## 5. Adaptive NMS

> 来源：Liu et al., "Adaptive NMS", CVPR 2019

### 5.1 核心思想

根据**局部密度**自适应调整 IoU 阈值。

```
密集区域:                     稀疏区域:
阈值调低 -> 允许更多重叠        阈值正常 -> 标准抑制
+---+---+---+                  +---+
| A | B | C |                  | A |
+---+---+---+                  +---+
                                  +---+
                                  | B |
                                  +---+
每个框都很近,                    距离远, 不需要
需要保留更多                     特殊处理
```

### 5.2 公式

```
Nt_adaptive = max(Nt, density_i)

其中 density_i 是框 b_i 周邻域的密度估计
```

## 6. NMS-Free：YOLOv10 双重分配

> 来源：Wang et al., "YOLOv10", 2024

### 6.1 核心思想

通过**训练时的双重标签分配**，在推理时完全消除 NMS。

```
传统 YOLO:                    YOLOv10:
训练 -> 多对一分配              训练 -> 一对一分配 (O2O)
推理 -> NMS 后处理              推理 -> 直接输出, 无需 NMS
延迟: +1~3ms                   延迟: 0ms
```

### 6.2 双重分配策略

```
训练阶段:
  One-to-Many (O2M) 分支: 每个 GT 匹配多个预测 (一对多)
    -> 提供丰富监督信号, 类似传统 YOLO

  One-to-One (O2O) 分支: 每个 GT 只匹配一个预测 (一对一)
    -> 推理时使用此分支输出

推理阶段:
  仅使用 O2O 分支输出
  每个目标最多一个预测框 -> 无需 NMS
```

### 6.3 优势与代价

| 维度 | NMS-Based | NMS-Free (YOLOv10) |
|------|-----------|-------------------|
| 推理延迟 | +1~3ms (NMS) | 0ms |
| 训练复杂度 | 标准 | 高（双重分支） |
| 密集场景 | 受 NMS 阈值影响 | 天然适配 |
| mAP | 略高（O2M 丰富监督） | 略低（约 -0.5 mAP） |

## 7. 后处理技术对比总结

| 技术 | 核心机制 | 密集场景表现 | 推理延迟 | 实现复杂度 | 典型应用 |
|------|---------|------------|---------|-----------|---------|
| Standard NMS | 硬阈值删除 | 差 | 低 | 低 | YOLOv3~v8 默认 |
| Soft-NMS | 置信度衰减 | 较好 | 低 | 低 | 密集检测 |
| DIoU-NMS | 中心距离抑制 | 好 | 低 | 中 | YOLOv5 可选 |
| WBF | 加权融合 | 好 | 高 | 中 | 模型集成/TTA |
| Adaptive NMS | 密度自适应阈值 | 好 | 中 | 高 | 密集行人检测 |
| NMS-Free | 训练时一对一分配 | 好 | 最低 | 高 | YOLOv10 |

## 8. 密集场景性能对比

以 CrowdHuman 数据集为例（密集行人检测）：

| 方法 | AP50 | Recall | 误删率 |
|------|------|--------|-------|
| Standard NMS (IoU=0.5) | 85.2% | 78.3% | 12.1% |
| Soft-NMS (sigma=0.5) | 87.1% | 82.6% | 7.5% |
| DIoU-NMS | 87.8% | 83.1% | 6.9% |
| WBF | 88.3% | 84.5% | 5.2% |
| Adaptive NMS | 88.5% | 85.0% | 4.8% |

> 注：以上数据为示意性对比，实际数值因模型和配置而异。

## 9. 在 Ultralytics 中的配置

```python
from ultralytics import YOLO

model = YOLO("yolov8n.pt")

# 标准 NMS 参数
results = model.predict(
    source="image.jpg",
    conf=0.25,          # 置信度阈值
    iou=0.45,           # NMS IoU 阈值
    max_det=300,        # 最大检测数
    agnostic_nms=False, # 类别无关 NMS
)

# 自定义后处理示例
import torchvision.ops as ops

# Soft-NMS
boxes = ops.nms(boxes, scores, iou_threshold=0.5)  # 替换为 soft_nms

# WBF (需要 wbf 库)
from ensemble_boxes import weighted_boxes_fusion
boxes, scores, labels = weighted_boxes_fusion(
    boxes_list, scores_list, labels_list,
    iou_thr=0.55, skip_box_thr=0.001
)
```
