# SAHI 切片推理：大图小目标的终极武器

> **阅读时间**: ~15 分钟 | **前置知识**: 目标检测推理流程、NMS 后处理、小目标检测挑战

SAHI（Slicing Aided Hyper Inference）是一种针对大尺寸图像中小目标检测的推理框架。其核心思想是：将大图切成多个重叠的小块（slice），对每个小块独立推理，再将所有小块的检测结果合并回原图坐标。SAHI 与模型无关，可搭配任何目标检测器使用。

## 1. SAHI 原理

### 1.1 为什么需要切片推理

标准推理流程中，输入图像被缩放到固定尺寸（如 640x640）。对于高分辨率大图（如 4000x3000 的航拍图或卫星图），缩放会导致小目标丢失大量像素信息：

```
原始大图 (4000x3000):
┌────────────────────────────────────────┐
│                                        │
│     🚗 (20x20 像素)                    │
│                                        │
│                                        │
└────────────────────────────────────────┘

缩放至 640x640 后:
  🚗 仅剩 ~3x3 像素 → 几乎不可检测

SAHI 切片 (512x512, 有重叠):
┌────────┬────────┬────────┐
│ 切片 1 │ 切片 2 │ 切片 3 │
│        │   🚗   │        │
├────────┼────────┼────────┤
│ 切片 4 │ 切片 5 │ 切片 6 │
└────────┴────────┴────────┘
  每个切片内, 🚗 保持 ~15x15 像素 → 容易检测
```

### 1.2 工作流程

```
步骤 1: 切片 (Slicing)
  原图 (W x H) ──> N 个重叠切片 (slice_height x slice_width)

步骤 2: 独立推理 (Inference)
  每个切片 ──> 检测模型 ──> 局部检测结果

步骤 3: 坐标映射 (Mapping)
  切片坐标系 ──> 原图坐标系 (加上切片偏移量)

步骤 4: 合并 (Postprocess)
  所有切片的检测结果 ──> NMS / GREEDYNMM ──> 最终结果
```

### 1.3 切片过程详解

```python
# SAHI 切片伪代码
def slice_image(image, slice_h, slice_w, overlap_h, overlap_w):
    H, W = image.shape[:2]
    step_h = int(slice_h * (1 - overlap_h))  # 垂直步长
    step_w = int(slice_w * (1 - overlap_w))  # 水平步长

    slices = []
    for y in range(0, H, step_h):
        for x in range(0, W, step_w):
            # 边界处理: 不超出图像范围
            y_end = min(y + slice_h, H)
            x_end = min(x + slice_w, W)
            y_start = max(y_end - slice_h, 0)
            x_start = max(x_end - slice_w, 0)

            slice_img = image[y_start:y_end, x_start:x_end]
            slices.append((slice_img, x_start, y_start))

    return slices  # (图像块, x偏移, y偏移)
```

## 2. 核心参数详解

### 2.1 参数总览

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `slice_height` | int | 512 | 切片高度（像素） |
| `slice_width` | int | 512 | 切片宽度（像素） |
| `overlap_height_ratio` | float | 0.2 | 垂直方向重叠率 (0.0~0.5) |
| `overlap_width_ratio` | float | 0.2 | 水平方向重叠率 (0.0~0.5) |
| `postprocess_type` | str | "NMS" | 后处理类型: "NMS" 或 "GREEDYNMM" |
| `postprocess_match_threshold` | float | 0.5 | 合并时的 IoU 匹配阈值 |
| `postprocess_class_agnostic` | bool | False | 是否跨类别合并 |

### 2.2 slice_height / slice_width: 切片大小

切片大小决定了每个推理子图的分辨率。

```
切片大小的影响:

512x512:
  - 切片数量多 → 推理次数多 → 慢
  - 小目标在切片中占比高 → 检测效果好

1024x1024:
  - 切片数量少 → 推理次数少 → 快
  - 小目标在切片中占比低 → 检测效果可能下降

选择原则:
  切片大小 ≥ 模型训练输入尺寸
  例: YOLOv8 训练尺寸 640 → 切片建议 640~1024
```

### 2.3 overlap_height_ratio / overlap_width_ratio: 重叠率

重叠率是 SAHI 最关键的参数之一，它决定了相邻切片之间的重叠程度。

```
重叠率 = 0.0 (无重叠):
┌────────┬────────┬────────┐
│ 切片 1 │ 切片 2 │ 切片 3 │
│        │        │        │
└────────┴────────┴────────┘
  问题: 位于切片边界的目标可能被截断 → 检测失败

重叠率 = 0.25 (25% 重叠):
┌──────────┐
│ 切片 1   │
│    ┌─────┼──────┐
│    │ 重叠│      │
└────┼─────┘      │
     │   切片 2   │
     └────────────┘
  目标至少在 1 个完整切片中 → 检测率提升

重叠率 = 0.5 (50% 重叠):
  - 每个目标出现在 4 个切片中
  - 检测率最高, 但切片数量翻倍 → 计算开销大
```

**重叠率选择建议**:

| 重叠率 | 切片数量倍增 | 边界目标保护 | 推荐场景 |
|--------|------------|-------------|---------|
| 0.0 | 基准 | 无 | 目标远离边界时 |
| 0.1 | ~1.1x | 弱 | 速度优先 |
| 0.2 | ~1.3x | 中等 | **通用推荐** |
| 0.3 | ~1.6x | 强 | 目标密集场景 |
| 0.5 | ~2.5x | 最强 | 竞赛/不计成本 |

### 2.4 postprocess_type: 合并策略

SAHI 提供两种后处理策略:

**NMS (Non-Maximum Suppression)**:
```
标准 NMS 流程:
  1. 按置信度排序所有检测框
  2. 选择最高分框
  3. 删除与其 IoU > 阈值的其他框
  4. 重复直到所有框处理完毕

优点: 快速, 广泛使用
缺点: 对重叠区域的重复框处理较粗糙
```

**GREEDYNMM (Greedy Non-Maximum Merging)**:
```
GREEDYNMM 流程:
  1. 按置信度排序
  2. 对匹配的框不直接删除, 而是合并 (加权平均)
  3. 合并后的框保留最高置信度

优点: 更好地利用重叠区域的多轮检测信息
缺点: 稍慢, 需要调参
```

### 2.5 postprocess_match_threshold: 匹配阈值

控制后处理时两个框被认为是同一目标的最大 IoU 阈值。

```
阈值 = 0.3: 严格匹配
  - 仅高度重叠的框才被合并
  - 可能漏掉边界处的重复检测

阈值 = 0.5: 适中 (默认推荐)

阈值 = 0.8: 宽松匹配
  - 轻微重叠的框也会被合并
  - 可能错误合并相邻的不同目标
```

## 3. 切片策略对小目标的影响

### 3.1 小目标的定义与挑战

COCO 定义: 面积 < 32x32 像素的目标为小目标。

在高分辨率大图中，小目标面临的核心问题是**信息损失**:

```
原图中 20x20 像素的行人:

标准推理 (缩放至 640x640):
  原图 4000x3000 → 缩放 6.25x → 目标仅 3x3 像素
  信息损失: ~97%

SAHI 切片 (slice=640, overlap=0.2):
  每个切片 640x640 → 目标在切片中约 15x15 像素
  信息损失: ~44%

SAHI 切片 (slice=512, overlap=0.2):
  每个切片 512x512 → 目标在切片中约 19x19 像素
  信息损失: ~9%
```

### 3.2 切片大小与目标尺寸的关系

| 切片大小 | 原图缩放比 | 20px 目标剩余 | 检测难度 |
|---------|-----------|-------------|---------|
| 无切片 (640) | 6.25x | ~3px | 极难 |
| 1024x1024 | 3.9x | ~5px | 难 |
| 640x640 | 1.0x | ~20px | 容易 |
| 512x512 | 1.0x | ~20px | 容易 |

**关键结论**: SAHI 的核心价值在于保持小目标在推理时的像素分辨率，切片越小，目标占比越高，但切片数量也越多。

### 3.3 重叠率对边界目标的影响

位于切片边界的目标可能被截断，重叠率越高，被完整覆盖的概率越大:

```
目标恰好在两个切片的边界上:

重叠率 = 0.0:
  切片 A: [目标左半] | 切片 B: [目标右半]
  两个切片都无法完整检测 → 漏检或低置信度

重叠率 = 0.2:
  切片 A: [目标完整]  ← 重叠区域覆盖了目标
  切片 B: [目标完整]
  至少有 1 个切片完整包含目标 → 正确检测
```

## 4. 与标准推理的对比

### 4.1 mAP 提升

SAHI 在多个基准数据集上的表现:

| 数据集 | 模型 | 标准推理 mAP | SAHI 推理 mAP | 提升 |
|--------|------|-------------|--------------|------|
| VisDrone | YOLOv8m | 39.2 | 47.3 | +8.1 |
| VisDrone | YOLOv8l | 41.5 | 49.8 | +8.3 |
| DOTA | YOLOv8m | 52.1 | 58.7 | +6.6 |
| 自定义航拍 | YOLOv8s | 35.6 | 44.2 | +8.6 |

### 4.2 时间开销

SAHI 的推理时间取决于切片数量:

```
切片数量估算:
  N = ceil(H / step_h) x ceil(W / step_w)
  其中 step = slice_size x (1 - overlap_ratio)

示例: 4000x3000 图像, slice=640, overlap=0.2
  step = 640 x 0.8 = 512
  N_h = ceil(3000/512) = 6
  N_w = ceil(4000/512) = 8
  总切片数 = 6 x 8 = 48

  推理时间 = 48 x 单切片推理时间 + 切片/合并开销
           = 48 x 8ms + 5ms
           = 389ms (约 2.6 FPS)
```

### 4.3 综合对比

| 维度 | 标准推理 | SAHI 推理 |
|------|---------|----------|
| mAP (小目标) | 基线 | +5~15% |
| mAP (大目标) | 基线 | 基本不变或略降 |
| 推理时间 | 基线 | 5~50x |
| 显存占用 | 基线 | 基本不变 (逐片推理) |
| 实现难度 | 简单 | 简单 (SAHI 库封装好) |
| 适用场景 | 通用 | 航拍/卫星/大图小目标 |

## 5. 参数调优指南

### 5.1 调优流程

```
推荐调优顺序:

1. 确定 slice 大小 (基于目标尺寸和模型输入)
   |
2. 确定 overlap_ratio (从 0.2 开始)
   |
3. 选择 postprocess_type (NMS 或 GREEDYNMM)
   |
4. 调整 postprocess_match_threshold (0.3~0.7)
   |
5. 验证效果, 回调 step 2~4
```

### 5.2 切片大小选择

```
决策树:

图像中最小目标 < 16px 像素?
  ├── 是 → slice = 320~512 (需要较大的放大倍率)
  └── 否 → 目标 < 32px?
              ├── 是 → slice = 512~640
              └── 否 → slice = 640~1024

注意: slice 大小应 ≥ 模型训练输入尺寸
```

### 5.3 重叠率选择

```
目标是否经常出现在图像的规则排列中 (如航拍中的车辆)?
  ├── 是 → overlap = 0.3~0.5 (需要更高重叠保护边界目标)
  └── 否 → 目标分布是否稀疏?
              ├── 是 → overlap = 0.1~0.2 (节省计算)
              └── 否 → overlap = 0.2 (默认推荐)
```

### 5.4 常见问题与解决方案

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| 边界目标漏检 | 重叠率不足 | 增大 overlap_ratio |
| 同一目标重复检测 | NMS 阈值过松 | 降低 postprocess_match_threshold |
| 相邻目标被合并 | NMS 阈值过紧 | 增大 postprocess_match_threshold 或使用 GREEDYNMM |
| 推理时间过长 | 切片过多 | 增大 slice 大小或减小 overlap |
| 大目标检测下降 | 切片太小 | 增大 slice 大小 |
| 显存溢出 | 切片太大 | 减小 slice 大小或使用逐片推理 |

## 6. SAHI + TTA 组合使用

SAHI 和 TTA 可以组合使用以进一步提升精度，但计算开销极大。

### 6.1 组合方式

```
方式 1: 先切片, 再对每个切片做 TTA
  原图 → SAHI 切片 → 每个切片做翻转 TTA → 合并

  推理次数 = 切片数 x TTA 倍数
  示例: 48 个切片 x 2 (翻转) = 96 次推理

方式 2: 先 TTA, 再对每个增强图做切片
  原图 → 翻转 → SAHI 切片 → 推理 → 合并

  两种方式效果接近, 方式 1 实现更简单
```

### 6.2 推荐配置

| 配置 | 切片 | TTA | 推理次数 | mAP 提升 | 适用场景 |
|------|------|-----|---------|---------|---------|
| 仅 SAHI | 640, overlap=0.2 | 无 | N | +6~10% | 日常使用 |
| SAHI + 翻转 | 640, overlap=0.2 | 翻转 | 2N | +8~12% | 论文报告 |
| SAHI + 多尺度 | 512/640/800 | 无 | 3N | +10~14% | 竞赛提交 |
| SAHI + 翻转 + 多尺度 | 512/640/800 | 翻转 | 6N | +12~16% | 极限精度 |

### 6.3 显存管理

SAHI + TTA 组合时显存管理尤为重要:

```python
# 推荐: 逐切片推理, 不要一次性加载所有切片
for slice_img, x_offset, y_offset in slices:
    # 单个切片推理 (显存占用恒定)
    results = model.predict(slice_img)
    # 立即处理结果, 释放显存
    all_results.extend(map_to_original(results, x_offset, y_offset))
```

## 7. VisDrone 上的 SAHI 实验

### 7.1 实验配置

- 数据集: VisDrone2019-DET val 集
- 模型: YOLOv8m (预训练 + VisDrone 微调)
- 原始图像尺寸: ~2000x1500
- 评估指标: mAP@0.5, mAP@0.5:0.95

### 7.2 结果对比

| 配置 | slice | overlap | mAP@0.5 | mAP@0.5:0.95 | 推理时间/帧 | FPS |
|------|-------|---------|---------|--------------|------------|-----|
| 标准推理 (640) | - | - | 39.2 | 23.8 | 8.5 ms | 117 |
| SAHI | 640 | 0.2 | 45.8 | 29.4 | 145 ms | 6.9 |
| SAHI | 512 | 0.2 | 47.3 | 30.8 | 210 ms | 4.8 |
| SAHI | 512 | 0.3 | 47.9 | 31.2 | 285 ms | 3.5 |
| SAHI + 翻转 TTA | 512 | 0.2 | 48.5 | 31.9 | 425 ms | 2.4 |
| SAHI + 3 尺度 TTA | 512 | 0.2 | 49.2 | 32.5 | 640 ms | 1.6 |

### 7.3 分类别分析

| 类别 | 标准推理 AP | SAHI AP | 提升 | 原因分析 |
|------|-----------|---------|------|---------|
| pedestrian | 32.1 | 42.5 | +10.4 | 行人尺寸小, SAHI 增益最大 |
| car | 55.3 | 60.1 | +4.8 | 车辆较大, 增益中等 |
| van | 28.7 | 34.2 | +5.5 | 中等尺寸 |
| bus | 42.1 | 45.3 | +3.2 | 大目标, 增益较小 |
| truck | 35.6 | 39.8 | +4.2 | 中等尺寸 |
| bicycle | 22.4 | 31.8 | +9.4 | 小目标, 增益大 |
| motorcycle | 25.3 | 34.1 | +8.8 | 小目标, 增益大 |

### 7.4 关键发现

1. **小目标增益最大**: pedestrian 和 bicycle/motorcycle 的 AP 提升超过 9 个百分点
2. **大目标增益有限**: bus 等大目标仅提升 3 个百分点，说明 SAHI 的价值主要在小目标
3. **重叠率 0.3 优于 0.2**: 在 VisDrone 的密集场景中，更高的重叠率减少了边界漏检
4. **SAHI + TTA 收益递减**: 在 SAHI 基础上叠加 TTA 的边际收益低于单独使用 TTA
