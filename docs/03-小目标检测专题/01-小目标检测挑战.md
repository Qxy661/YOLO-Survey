## 1. 什么是小目标？

不同数据集对"小目标"的定义略有不同，业界通常以**像素面积**作为衡量标准。

### 1.1 COCO 定义

MS COCO 数据集将目标按面积分为三个等级：

| 等级 | 英文名 | 面积阈值 | 直观尺寸 | 占比（约） |
|------|--------|----------|----------|-----------|
| 小目标 | Small | area < 32^2 = 1024 px^2 | < 32 x 32 像素 | ~41% |
| 中目标 | Medium | 32^2 <= area < 96^2 | 32~96 像素 | ~34% |
| 大目标 | Large | area >= 96^2 | > 96 像素 | ~25% |

> **注意**: 这里的 area 指的是分割掩码 (segmentation mask) 的像素面积，而非 bounding box 面积。

### 1.2 VisDrone 定义

VisDrone 数据集来自**无人机视角 (UAV perspective)**，由于拍摄高度远高于地面，即使是真实世界中的大型物体（如汽车），在图像中也可能只有十几个像素。其小目标标准更为严苛：

| 场景 | 典型目标尺寸 | 说明 |
|------|-------------|------|
| 高空悬停 (> 100m) | 8 x 8 ~ 20 x 20 px | 几乎只有几个像素点 |
| 中等高度 (50~100m) | 20 x 20 ~ 50 x 50 px | 勉强可辨轮廓 |
| 低空飞行 (< 50m) | 50 x 50 ~ 150 x 150 px | 接近常规目标尺寸 |

### 1.3 小目标的直观感受

```
  常规目标 (COCO)            无人机视角小目标 (VisDrone)
  ┌──────────────────┐       ┌──────────────────────────────────────┐
  │                  │       │                                      │
  │   ┌──────────┐   │       │                                      │
  │   │ ████████ │   │       │                                      │
  │   │ ████████ │   │       │         . .                          │
  │   │ ████████ │   │       │        . . .     .                   │
  │   │ ████████ │   │       │         .          . .               │
  │   └──────────┘   │       │                        . .           │
  │     ~200x200px   │       │       .                              │
  │                  │       │            .    .                     │
  │                  │       │         .     (~10x10px per dot)      │
  └──────────────────┘       └──────────────────────────────────────┘
   信息丰富，特征清晰            信息极少，难以辨识
```

## 2. 小目标检测为什么难？

小目标检测是目标检测领域公认的难题。以下从四个核心维度分析其困难根源。

### 2.1 特征信息少：经过多次下采样后消失

这是小目标检测**最根本的挑战**。主流检测网络（如 YOLO、ResNet backbone）通过逐步下采样来提取高层语义特征，但小目标在这个过程中会丢失大量甚至全部信息。

```
输入图像 (640x640)     下采样过程中小目标的命运：

  原图中一个              P3 (80x80)    P4 (40x40)    P5 (20x20)
  12x12 像素目标    ──►  占 1x1 格     几乎消失      完全消失
  ┌──┐
  │██│ 12px               ■              .             (无)
  └──┘

  一个 8x8 像素目标  ──►  不足 1 格     消失          消失
  ┌──┐
  │█ │  8px              (无)           (无)           (无)
  └──┘
```

**下采样过程量化分析**（以 640x640 输入为例）：

| 层级 | Feature Map 尺寸 | 步长 (Stride) | 可检测最小目标 | 小目标 (< 32px) 状态 |
|------|-----------------|---------------|---------------|---------------------|
| P3 | 80 x 80 | 8 | ~16 px | 勉强保留 1~2 个格子 |
| P4 | 40 x 40 | 16 | ~32 px | 基本消失 |
| P5 | 20 x 20 | 32 | ~64 px | 完全消失 |
| P6 (部分网络) | 10 x 10 | 64 | ~128 px | 不可能检测 |

**关键结论**: 一个小目标如果在原图中只有 12x12 像素，经过 stride=8 的下采样后，在 P3 层的 feature map 上仅占据约 1.5 个格子的信息量，到 P4 层就几乎消失了。这正是 FPN (Feature Pyramid Network) 被提出的重要原因之一——通过自顶向下的特征融合来增强浅层的语义信息。

### 2.2 定位精度要求高：像素级误差导致 IoU 大幅下降

小目标的 bounding box 面积很小，因此即使是**1~2 个像素的定位偏差**，也会导致 IoU (Intersection over Union) 急剧下降，从而漏检。

```
  大目标 (200x200) 的定位误差影响：

  预测框偏移 5px          IoU = (195x195) / (205x205) ≈ 0.90
  ┌─────────────────┐     影响很小，依然算正确检测
  │ ┌───────────────┤
  │ │   GT Box      │
  │ │               │
  └─┤               │
    └───────────────┘

  小目标 (10x10) 的定位误差影响：

  预测框偏移 5px          IoU = (5x5) / (15x15) ≈ 0.11
  ┌─────────┐             远低于阈值 0.5，判定为漏检！
  │ ┌─────┐ │
  └─│ GT  │─┘
    └─────┘
```

**IoU 随定位误差变化的量化对比**：

| 目标尺寸 | 偏移 1px IoU | 偏移 3px IoU | 偏移 5px IoU | 偏移 10px IoU |
|----------|-------------|-------------|-------------|--------------|
| 10 x 10 | 0.82 | 0.49 | 0.25 | 0.00 |
| 20 x 20 | 0.90 | 0.74 | 0.56 | 0.25 |
| 50 x 50 | 0.96 | 0.89 | 0.82 | 0.64 |
| 100 x 100 | 0.98 | 0.94 | 0.91 | 0.82 |
| 200 x 200 | 0.99 | 0.97 | 0.95 | 0.91 |

> **结论**: 对于 10x10 的小目标，仅 3 像素的定位偏差就会使 IoU 降至 0.49，低于常用阈值 0.5，导致漏检。而对于 200x200 的大目标，10 像素的偏差仍可保持 0.91 的高 IoU。

### 2.3 类别不平衡：小目标数量多但占比小

在大多数数据集中，小目标的**实例数量占比很高**，但**面积占比很低**，导致训练时梯度被大目标主导。

```
  训练样本中目标面积分布（示意）：

  面积占比                          实例数量占比
  ┌──────────────────────┐         ┌──────────────────────┐
  │████████████ 大目标    │ 55%     │██ 小目标              │ 15%
  │█████ 中目标           │ 35%     │██████ 中目标          │ 35%
  │██ 小目标              │ 10%     │████████████ 大目标    │ 50%
  └──────────────────────┘         └──────────────────────┘
   小目标面积占比极低                 小目标数量占比最高
```

这种不平衡带来两个问题：

1. **Loss 不平衡**: 小目标的 anchor 数量少但负样本多，大量背景 anchor 产生的 loss 淹没了小目标的有效梯度
2. **NMS 困难**: 小目标密集区域，大量预测框互相重叠，NMS (Non-Maximum Suppression) 容易误删正确检测

### 2.4 尺度变化大：同一物体远近不同

无人机拍摄场景中，由于飞行高度和距离变化，**同一类别的目标尺度差异巨大**：

```
  无人机拍摄场景中的尺度变化：

  高空区域                              低空区域
  ┌─────────────────────────────────────────────────────┐
  │   . .  .                    ┌──────────┐            │
  │  . . .    .                 │ ████████ │            │
  │    .   .                    │ ████████ │            │
  │         . .                 │ ████████ │            │
  │  .    .                     └──────────┘            │
  │       .                   同一类别的车：             │
  │  (~10px 小汽车)              (~150px 大汽车)         │
  └─────────────────────────────────────────────────────┘
         尺度比可达 1:15 甚至更大
```

尺度变化对检测器的挑战：

| 挑战 | 具体表现 |
|------|---------|
| Anchor 设计困难 | 需要覆盖从 8px 到 200px 的范围，预设 anchor 尺寸难以兼顾 |
| 特征尺度不匹配 | 单一尺度的 feature map 无法同时检测极大和极小目标 |
| 训练目标冲突 | 大目标和小目标对特征提取的需求相互矛盾 |

## 3. VisDrone 数据集特点

[VisDrone](https://github.com/VisDrone/VisDrone-Dataset) 是目前最主流的无人机视角目标检测基准数据集之一，由天津大学 AISKYEYE 团队收集标注。

### 3.1 数据集概览

| 属性 | 值 |
|------|-----|
| 图片数量 | 10,209 张 (train: 6,471 / val: 548 / test-dev: 1,610 / test-challenge: 1,580) |
| 视频序列 | 96 个片段 |
| 图片分辨率 | 约 2000 x 1500 像素 |
| 拍摄平台 | 无人机 (多旋翼) |
| 拍摄高度 | 5m ~ 800m |
| 标注类别 | 10 类 + 1 类 (ignored region) |
| 标注格式 | Bounding Box (无实例分割) |
| 总标注数 | ~380,000 个目标实例 |

### 3.2 十个类别

| 编号 | 类别 | 英文 | 说明 |
|------|------|------|------|
| 0 | 行人 | pedestrian | 步行的人 |
| 1 | 人 (骑手) | person | 骑车/电动车上的人 |
| 2 | 自行车 | bicycle | 包括电动车 |
| 3 | 汽车 | car | 最常见的类别 |
| 4 | 厢式货车 | van | 中型车辆 |
| 5 | 卡车 | truck | 大型车辆 |
| 6 | 三轮车 | tricycle | 三轮摩托车 |
| 7 | 遮阳三轮车 | awning-tricycle | 有遮阳棚的三轮车 |
| 8 | 公共汽车 | bus | 大巴 |
| 9 | 摩托车 | motorcycle | 两轮摩托车 |

## 4. 小目标检测关键挑战总结

下表从多个维度系统总结小目标检测面临的核心挑战及其对检测性能的影响：

| 挑战维度 | 具体问题 | 对检测的影响 | 严重程度 | 现有缓解方案 |
|----------|---------|-------------|---------|-------------|
| **特征信息不足** | 下采样后特征消失 | 小目标无响应，直接漏检 | ★★★★★ | FPN, PANet, 高分辨率检测头 |
| **定位精度** | 微小偏移导致 IoU 骤降 | 检测框不准，误判为漏检 | ★★★★★ | 细粒度回归, Point-based 检测 |
| **类别不平衡** | 小目标样本少/梯度弱 | 模型偏向大目标 | ★★★★☆ | Focal Loss, 过采样, 数据增强 |
| **尺度变化** | 同类目标尺寸差异大 | 单一尺度检测器失效 | ★★★★☆ | 多尺度训练, 自适应 anchor |
| **密集遮挡** | 目标互相遮挡 | NMS 误删, 边界混淆 | ★★★★☆ | Soft-NMS, NMS-Free 检测器 |
| **背景干扰** | 小目标与背景难区分 | 大量误检 (False Positive) | ★★★☆☆ | 注意力机制, 上下文建模 |
| **标注质量** | 小目标标注框不准 | 训练标签噪声 | ★★★☆☆ | 高倍率标注, 辅助标注工具 |
| **计算开销** | 高分辨率输入代价大 | 推理速度慢 | ★★★☆☆ | 轻量化高分辨率, Tile 策略 |

### 5.3 关键发现

1. **行人和骑手是最小的目标**: 中位面积仅 400~520 px^2（约 20~23px 边长），超过 80% 的实例属于小目标

## 思考题

<details>
<summary>1. 为什么不能简单地通过放大输入图像来解决小目标检测问题？</summary>

直接放大图像确实可以增大目标的像素面积，但会带来以下问题：

- **计算量爆炸**: 输入尺寸翻倍，FLOPs 约增加 4 倍，推理时间大幅增加
- **大目标退化**: 原本的大目标可能超出检测器的有效感受野范围
- **内存限制**: GPU 显存有限，无法无限制增大输入
- **更优方案**: 多尺度特征融合 (FPN)、Tile 切片检测、多尺度训练等方法更高效
</details>

<details>
<summary>2. FPN 是如何缓解小目标检测问题的？它有哪些局限性？</summary>

FPN (Feature Pyramid Network) 通过自顶向下的路径和横向连接，将高层语义特征与浅层高分辨率特征融合，使浅层 feature map 也具备丰富的语义信息。

**局限性**：
- 特征融合过程可能引入噪声，浅层的定位信息和高层的语义信息并不总是互补
- 融合后的特征仍受限于原始浅层 feature map 的分辨率
- 对于极小目标（< 8px），即使在 P3 层也难以提供足够信息
- 后续的 PANet、BiFPN、NAS-FPN 等工作在此基础上进一步优化了融合策略
</details>

<details>
<summary>3. 在 VisDrone 数据集中，为什么 "car" 类别虽然实例数量最多，但检测难度并非最大？</summary>

虽然 car 的实例数量最多（~40%），但：

- 车辆的**中位面积** (~1400 px^2) 远大于行人 (~520 px^2)
- 车辆的**形状较为规整**，长宽比稳定，anchor 匹配更容易
- 车辆的**纹理特征**更丰富（车窗、车轮等），即使在较小尺寸下也较易识别
- 真正的难点类别是 **pedestrian** 和 **person**（骑手），它们面积最小、形变最大、遮挡最严重
</details>

<details>
<summary>4. 如果要设计一个针对无人机视角小目标检测的数据增强策略，你会考虑哪些方法？</summary>

可以考虑以下策略组合：

- **Mosaic / MixUp**: 将多张图拼接，增加小目标出现的上下文多样性
- **Copy-Paste Augmentation**: 将小目标从原图裁剪后粘贴到其他位置，直接增加小目标样本数
- **随机缩放**: 训练时随机缩放图像，让模型学习不同尺度
- **Tile 切片**: 将大图切成小块，每块中目标相对更大
- **过采样小目标图片**: 在训练集中更多地采样包含密集小目标的图片
- 需要注意避免过度增强导致训练不稳定或过拟合
</details>

<details>
<summary>5. Anchor-Free 检测器（如 FCOS、CenterNet）在小目标检测上有何优势和劣势？</summary>

**优势**：
- 不需要预设 anchor 尺寸，避免了 anchor 与小目标不匹配的问题
- 基于逐像素预测，天然适合密集小目标场景
- 参数量更少，训练更简单

**劣势**：
- 依赖于 feature map 上的**中心点定位**，小目标的中心点只有 1 个像素，召回率受限
- 对特征图分辨率要求更高，否则中心点可能在下采样中丢失
- FCOS 的 centerness 分支对小目标可能产生不稳定的预测

总体来说，Anchor-Free 方法在小目标检测上有一定潜力，但并非银弹，仍需结合多尺度策略使用。
</details>
