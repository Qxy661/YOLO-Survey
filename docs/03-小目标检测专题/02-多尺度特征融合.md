# 多尺度特征融合: 从 FPN 到自适应融合的演进

> **一句话总结**: 多尺度特征融合通过连接不同深度的特征图，使检测器同时具备
> 强语义理解和精准空间定位能力，是解决小目标检测的关键技术。
>
> 本文从 FPN 的基本原理出发，逐步介绍 PANet、BiFPN、PAFPN 等经典结构，
> 以及 ASFF、AAFM 等自适应融合方法，最后聚焦 P2 层对小目标检测的特殊意义。

## 1. 为什么需要多尺度特征融合

### 1.1 CNN 特征层次的固有差异

```
浅层特征 (P2, stride=4)          深层特征 (P5, stride=32)
  +-----------------------+        +-----------------------+
  | 高分辨率              |        | 低分辨率              |
  | 强空间定位能力        |        | 弱空间定位能力        |
  | 弱语义信息            |        | 强语义信息            |
  | 适合小目标            |        | 适合大目标            |
  +-----------------------+        +-----------------------+
```

### 1.2 单一尺度的局限

```
问题场景:
  输入图像 640x640，P5 特征图 20x20
  一个 10x10 像素的小目标，在 P5 上仅占 ~0.3x0.3 个像素
  几乎无法被检测到
```

**核心矛盾**: 语义信息和空间分辨率成反比，必须通过特征融合来兼顾两者。

## 2. FPN: 自顶向下特征金字塔 (2017)

**论文**: *Feature Pyramid Networks for Object Detection* (CVPR 2017, Lin 等)

### 2.1 结构与融合公式

```
自底向上 (Backbone)         自顶向下 (FPN)

C2 (160x160) ----+
                  |
C3 (80x80) ------+-----> lateral(1x1) + upsample + add --> P3 (80x80)
                  |                                           |
C4 (40x40) ------+-----> lateral(1x1) + upsample + add --> P4 (40x40)
                  |                                           |
C5 (20x20) ------+-----> lateral(1x1) -----------------> P5 (20x20)
```

```
FPN 单层融合: P_i = Conv_1x1(C_i) + Upsample(P_{i+1})

  Conv_1x1(C_i)     = 横向连接，统一通道数到 256
  Upsample(P_{i+1}) = 2x 最近邻上采样
  +                  = 逐元素相加
```

### 2.2 优缺点

| 优点 | 缺点 |
|------|------|
| 高层语义传递到低层 | 只有自顶向下一条路径 |
| 所有尺度有强语义 | 低层定位信息未回传高层 |
| 结构简单 | 大目标定位精度有限 |

## 3. PANet: 双向路径聚合 (2018)

**论文**: *Path Aggregation Network for Instance Segmentation* (CVPR 2018, Liu 等)

### 3.1 双向结构

```
                 FPN 路径 (自顶向下)     PANet 新增 (自底向上)

C2 -----> P2                                P2
           ^                                 |
C3 -----> P3                                 v
           ^                                 P3'
C4 -----> P4                                 |
           ^                                 v
C5 -----> P5                                 P5'

  自顶向下: 传递强语义信息到浅层
  自底向上: 传递精确定位信息到深层
```

### 3.2 与 FPN 对比

| 特性 | FPN | PANet |
|------|-----|-------|
| 融合方向 | 单向 | **双向** |
| 大目标检测 | 一般 | **更好** |
| 小目标检测 | 较好 | 较好 |
| 计算开销 | 基准 | +约 20% |

## 4. BiFPN: 加权双向特征金字塔 (2020)

**论文**: *EfficientDet: Scalable and Efficient Object Detection* (CVPR 2020, Tan 等)

### 4.1 三大改进

```
改进 1: 移除单输入节点 (无融合贡献的节点)
改进 2: 同一层添加跳跃连接 (保持信息流完整性)
改进 3: 可学习权重平衡各层贡献
```

### 4.2 加权融合公式

```
O = sum(w_i * I_i) / (sum(w_i) + epsilon)

快速归一化融合:
  O = (w1*I1 + w2*I2) / (w1 + w2 + eps)

优势:
  - 权重无约束 (可为任意正数)
  - 计算高效 (无需 softmax)
  - 梯度稳定
```

### 4.3 完整 BiFPN 结构

```
层级:       P7       P6       P5       P4       P3

原始输入:    P7_in    P6_in    P5_in    P4_in    P3_in
              |        |        |        |        |
              +<-------+        +<-------+        |
              |  P6_td |        |  P4_td |        |
              |        +<-------+        +<-------+
              |        |  P5_td |        |  P3_td |
              |        +------->+        +------->+
              +------->+        +------->+        |
输出:        P7_out   P6_out   P5_out   P4_out   P3_out
```

| 模型 | BiFPN 轮数 | BiFPN 通道数 | mAP@0.5:0.95 |
|------|-----------|-------------|---------------|
| EfficientDet-D0 | 3 | 64 | 33.8% |
| EfficientDet-D2 | 5 | 112 | 43.0% |
| EfficientDet-D4 | 7 | 224 | 49.4% |

## 5. PAFPN: YOLO 系列的标准配置

PAFPN (Path Aggregation FPN) 是 YOLOv5/v8 中实际采用的 Neck 结构:

```
YOLOv8 PAFPN:

  P5_in ──┬── Upsample ──┐
          |               |
  P4_in ──┼── Concat ──> C2f ──> P4_neck
          |                          |
  P3_in ──┴── Concat ──> C2f ──> P3_neck
                                      |
  P3_neck ── Conv(s=2) ──┬── Concat ──> C2f ──> P4_out
  P4_neck ────────────────┘                          |
  P4_out ── Conv(s=2) ──┬── Concat ──> C2f ──> P5_out
  P5_in ─────────────────┘
```

| 特性 | PANet (原版) | PAFPN (YOLO 实现) |
|------|-------------|-------------------|
| 融合方式 | 逐元素相加 | **Concat + Conv** |
| 融合模块 | 简单卷积 | **C2f 模块** |
| 通道压缩 | 无 | 通过 C2f 控制 |

## 6. 自适应融合方法

### 6.1 ASFF (Adaptively Spatial Feature Fusion)

```
对每个空间位置 (i,j) 自适应学习三个尺度的融合权重:

  Output(i,j) = alpha(i,j)*P3(i,j) + beta(i,j)*P4(i,j) + gamma(i,j)*P5(i,j)
  其中 alpha + beta + gamma = 1 (softmax 约束)
```

### 6.2 AAFM (Attention-Augmented Feature Module)

```
多尺度特征 {P2, P3, P4, P5}
        |
   通道注意力 (SE/ECA)  ── 学习"哪些通道"更重要
        |
   空间注意力 (CBAM)    ── 学习"哪些位置"更重要
        |
   加权融合 ──> 输出
```

### 6.3 自适应方法对比

| 方法 | 融合粒度 | 计算开销 | 精度提升 |
|------|---------|---------|---------|
| FPN (相加) | 全局 | 低 | 基准 |
| BiFPN (加权) | **层级** | 低 | +1~2 mAP |
| ASFF (像素级) | **像素级** | 中 | +1~2 mAP |
| AAFM (注意力) | **通道+空间** | 中高 | +1~3 mAP |

## 7. 方法对比总结

| 方法 | 年份 | 融合方向 | 融合方式 | 加权机制 | 代表模型 |
|------|------|---------|---------|---------|---------|
| FPN | 2017 | 自顶向下 | 相加 | 无 | Faster R-CNN |
| PANet | 2018 | **双向** | 相加 | 无 | Mask R-CNN |
| BiFPN | 2020 | 双向+跳跃 | 相加 | **可学习** | EfficientDet |
| PAFPN | 2020 | 双向 | Concat | 无 | YOLOv5/v8 |
| ASFF | 2019 | 多尺度 | 加权 | **像素级** | YOLOv3-ASFF |

**演进趋势**: 简单相加 → 双向聚合 → 可学习权重 → 空间自适应 → 注意力增强。

## 8. 小目标检测中的 P2 层意义

### 8.1 各层分辨率与小目标关系

```
层级   分辨率     stride   单像素覆盖    小目标(32px)占特征像素数
P2     160x160    4        4x4 像素      8x8 = 64 像素  <-- 信息充足
P3     80x80      8        8x8 像素      4x4 = 16 像素  <-- 较好
P4     40x40      16       16x16 像素    2x2 = 4 像素   <-- 勉强
P5     20x20      32       32x32 像素    1x1 = 1 像素   <-- 几乎丢失
```

### 8.2 代价与收益

```
启用 P2 检测头 (160x160 特征图):

代价: 计算量 +30~40%，推理速度下降，显存增加
收益: 小目标 mAP +3~5 个百分点
```

### 8.3 何时需要 P2 层

| 场景 | 是否需要 | 原因 |
|------|---------|------|
| 无人机航拍 | **强烈建议** | 目标小且密集 |
| 卫星遥感 | **必须** | 目标极小 |
| 自动驾驶 (远距离) | 建议 | 远处目标较小 |
| 通用场景 (COCO) | 可选 | P3 通常足够 |

### 8.4 小目标检测综合策略

```
1. 启用 P2 检测头: 增加 160x160 高分辨率特征图
2. Mosaic / MixUp 增强: 提升小目标出现频率
3. 提高输入分辨率: 640 -> 1280 显著提升小目标 AP
4. 使用 DFL 损失: 分布式回归对小边界框更精确
5. SAHI 切片推理: 大图切分为小块分别检测
6. 注意力增强融合: C2PSA / A2 增强小目标特征响应
```

## 10. 思考题

<details>
<summary><strong>1. FPN 为什么使用逐元素相加而不是 Concat？</strong></summary>

逐元素相加保持通道数不变 (都映射到 256 通道后相加)，计算量更小且显存占用更低。Concat 会增加通道数，需要额外卷积来压缩。不过 YOLOv8 的 PAFPN 选择了 Concat，因为 C2f 模块本身就需要多通道输入来发挥优势。

</details>

<details>
<summary><strong>2. BiFPN 移除单输入节点的原因是什么？</strong></summary>

单输入节点只有一个特征输入，没有与其他层的融合发生，对特征融合没有实质贡献。移除后减少不必要的计算，同时用跳跃连接保持信息流完整性，既提高了效率又不损失精度。

</details>

<details>
<summary><strong>3. BiFPN 的可学习权重相比固定权重的优势在哪里？</strong></summary>

不同层的特征对最终融合的贡献是不均匀的。检测小目标时浅层特征更重要，检测大目标时深层特征更重要。可学习权重让网络根据任务自适应地调整各层的贡献比例，实验表明能带来 1-2 mAP 的提升。

</details>

<details>
<summary><strong>4. 在小目标检测中，为什么不能只用 P2 层而不用 P5 层？</strong></summary>

只用 P2 层会丢失高层的强语义信息。P2 层虽然分辨率高，但感受野小，缺乏对物体类别的高层理解。小目标检测需要同时具备高分辨率来保留空间细节 (P2/P3) 和强语义信息来正确分类 (P4/P5)。两者缺一不可。

</details>

<details>
<summary><strong>5. ASFF 的像素级权重和 BiFPN 的层级权重有什么本质区别？</strong></summary>

BiFPN 的权重是每个层级一个标量，同一层内所有空间位置共享。ASFF 的权重是每个空间位置独立的，即对于同一层的不同像素，融合三个尺度的权重各不相同。这使得 ASFF 能够在"前景"和"背景"区域使用不同的融合策略，对小目标密集场景更有效，但计算开销更大。

</details>
