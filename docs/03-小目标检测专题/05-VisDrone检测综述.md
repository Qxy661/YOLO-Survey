# VisDrone 无人机视角目标检测综述

VisDrone 是当前最具代表性的无人机视角目标检测基准数据集。其场景复杂、目标密集且尺度变化剧烈，已成为检验小目标检测算法能力的重要试金石。本文系统梳理 VisDrone 数据集特点、检测难点、历年竞赛方案以及基于 YOLO 的优化策略。

## 1. VisDrone 数据集概述

### 1.1 数据集背景

VisDrone（Vision Meets Drones）由天津大学 AISKYEYE 团队于 2018 年发布，拍摄角度为俯视/斜视，目标尺度普遍偏小，覆盖城市街道、乡村、高速公路等真实环境。

### 1.2 类别定义 (10 类)

| 编号 | 类别 | 中文 | 典型尺寸 | 难度 |
|------|------|------|----------|------|
| 0 | pedestrian | 行人 | 极小 (~12x24px) | 高 |
| 1 | people | 人群 | 极小 (~10x18px) | 高 |
| 2 | bicycle | 自行车 | 小 (~16x20px) | 高 |
| 3 | car | 轿车 | 中等 (~40x30px) | 中 |
| 4 | van | 面包车 | 中等 | 中 |
| 5 | truck | 卡车 | 较大 | 低 |
| 6 | tricycle | 三轮车 | 小 | 高 |
| 7 | awning-tricycle | 篷式三轮车 | 小 | 高 |
| 8 | bus | 公交车 | 大 | 低 |
| 9 | motor | 摩托车 | 小 (~14x18px) | 高 |

10 个类别中有 6 个属于小目标类别，VisDrone 天然适合检验小目标检测能力。

### 1.3 数据集划分

| 子集 | 图片数量 | 标注状态 |
|------|----------|----------|
| Train | 6,471 | 公开 |
| Val | 548 | 公开 |
| Test-dev | 1,610 | 不公开 |
| Test-challenge | 1,580 | 不公开 |
| **总计** | **10,209** | -- |

图片分辨率约 2000x1500，拍摄高度 10-80m，总标注框 500 万+。

### 1.4 标注格式

```
bbox_left, bbox_top, bbox_width, bbox_height, score, object_category, truncation, occlusion
```

其中 `truncation`（截断）和 `occlusion`（遮挡）是 VisDrone 独有的标注维度，为研究遮挡场景提供直接监督信号。

## 2. 检测难点分析

### 2.1 小目标密集

VisDrone 中约 **60%-70% 的目标属于小目标**（面积 < 32x32 px），远高于 COCO 的 ~42%。

```
目标高度分布 (像素):

  60% ┤ ██████████
      │ ██████████
  40% ┤ ██████████ ████████
      │ ██████████ ████████
  20% ┤ ██████████ ████████ ████
      │ ██████████ ████████ ████ ██
   0% ┼──────────────────────────────
      0-16px  16-32px  32-64px  >64px
       (小)    (中)     (中大)   (大)

约 50%+ 目标高度小于 32 像素
```

### 2.2 遮挡严重

| 遮挡/截断类型 | 占比 | 检测影响 |
|---------------|------|----------|
| 完全可见 | ~35% | 标准难度 |
| 部分遮挡 (occlusion=1) | ~25% | 特征不完整，易漏检 |
| 大部分遮挡 (occlusion=2) | ~10% | 极难检测 |
| 截断 (truncation>0.5) | ~15% | 边界框回归困难 |
| 组合情况 | ~15% | 最难场景 |

### 2.3 类别不平衡

```
car          ████████████████████  ~35-40%
pedestrian   ██████████            ~12-20%
people       █████                 ~5-10%
motor        ████                  ~5-8%
bicycle      ███                   ~3-6%
van          ██                    ~5%
truck        ██                    ~3%
tricycle     █                     ~2%
awning-tri   █                     ~1%
bus          █                     ~2%
```

car 与 awning-tricycle 的样本比约 **20:1**，长尾分布导致罕见类别 mAP 极低。

### 2.4 尺度变化大

同一张图片中近处车辆 200x200px，远处行人仅 8x15px，**尺度跨度超过 20 倍**。

### 2.5 其他挑战

| 挑战 | 表现 |
|------|------|
| 光照变化 | 阴影、逆光、夜间低照度 |
| 运动模糊 | 无人机飞行抖动 |
| 视角多样 | 正射俯视 vs 斜视 45 度 |
| 背景复杂 | 树木、建筑物干扰 |
| 标注噪声 | 部分小目标标注边界不一致 |

## 4. 代表性方法

### 4.1 ClusDet -- 聚类检测策略 (2018)

**论文**: *ClusDet: Cluster Detection in Drone Images*

**核心思想**: 将高分辨率图像分割为若干密集区域，对每个区域单独检测，避免直接缩放导致小目标信息丢失。

```
输入图像 (2000x1500)
    |
    v
Density Map Estimation (密度图估计)
    |
    v
Cluster Region Extraction (聚类区域提取)
    |
    v
对每个聚类区域独立检测 (保留原始分辨率)
    |
    v
全局检测 (低分辨率，捕获稀疏目标)
    |
    v
结果融合 + NMS
```

- **优势**: 避免大图直接缩放到 640x640 导致的小目标信息丢失
- **局限**: 聚类区域数量需预设，推理速度慢 (~3 FPS)
- **性能**: mAP@0.5 约 21.4%

### 4.2 DMNet -- 密度图引导检测 (2019)

**论文**: *Density Map Guided Object Detection in Aerial Images*

利用密度估计网络预测目标密集区域，自适应分配计算资源。与 ClusDet 的区别：根据密度图自适应调整分割，更灵活。

- **性能**: mAP@0.5 约 25.2%，密集场景优势明显

### 4.3 GLSAN -- 全局-局部注意力网络 (2020)

**论文**: *Global-Local Self-Attention Network for Drone Image Detection*

```
Backbone (ResNet-50)
    |
    +---> Global Branch (全局注意力) -- 捕获场景级语义
    |
    +---> Local Branch (局部注意力)  -- 聚焦小目标区域
    |
    v
Feature Fusion (自适应融合) --> Detection Head
```

- **性能**: mAP@0.5 约 29.3%，行人和自行车检测提升显著

### 4.4 QueryDet -- 查询式检测 (2021)

**论文**: *QueryDet: Query-based Transformer for Small Object Detection*

将 Transformer 的 query 机制引入小目标检测，通过可学习查询向量主动搜索小目标位置。

```
低分辨率特征 --> 粗糙检测 --> 提取小目标候选位置 (Queries)
                                    |
高分辨率特征 <-- 在候选位置处密集检测（只计算这些位置）
```

| 组件 | 说明 |
|------|------|
| Query Generation | 粗粒度特征图上生成候选查询位置 |
| Query Refinement | 细粒度特征图上精化查询 |
| Sparse Attention | 只在查询位置附近计算注意力 |

- **性能**: mAP@0.5 约 33.1%，推理速度比密集 Transformer 快约 2 倍

### 4.5 FocusNet -- 聚焦小目标 (2022)

**论文**: *FocusNet: Focusing on Small Objects for Drone Image Detection*

通过自适应特征选择和增强模块强制网络关注小目标区域：

- Small Object Enhancement Module (SOEM): 浅层特征上增强小目标响应
- Adaptive Feature Selection: 根据目标尺度自动选择最优特征层
- Hard Example Mining: 对漏检小目标进行困难样本挖掘

- **性能**: mAP@0.5 约 35.6%

### 4.6 方法对比总结

| 方法 | 年份 | 核心思路 | mAP@0.5 | FPS | 优势 | 局限 |
|------|------|----------|---------|-----|------|------|
| ClusDet | 2018 | 聚类分割 | 21.4% | ~3 | 保留原图分辨率 | 速度慢 |
| DMNet | 2019 | 密度引导 | 25.2% | ~5 | 自适应分割 | 需额外密度网络 |
| GLSAN | 2020 | 双分支注意力 | 29.3% | ~8 | 全局上下文 | 参数量大 |
| QueryDet | 2021 | 查询式稀疏 | 33.1% | ~12 | 计算高效 | 训练复杂 |
| FocusNet | 2022 | 小目标聚焦 | 35.6% | ~10 | 精准增强 | 依赖高质量标注 |

## 5. 基于 YOLO 的优化策略

YOLO 系列因速度优势是 VisDrone 实际部署的首选，但原版 YOLO 在 VisDrone 上表现不理想，需针对性优化。

### 5.1 输入分辨率

| 分辨率 | mAP@0.5 (YOLOv8-n) | FPS | 说明 |
|--------|---------------------|-----|------|
| 640 | ~18.5% | ~120 | 实时性要求高 |
| 1280 | ~26.3% | ~45 | **性价比最高** |
| 1920 | ~30.1% | ~18 | 收益递减 |

**关键结论**: 640 到 1280 提升约 8 个百分点，是最具性价比的单一优化手段。

### 5.2 P2 检测头

标准 YOLO 使用 P3/P4/P5 (80x80/40x40/20x20)。对极小目标 (<16px)，P3 的 8x 下采样仍然过大。

```
标准检测头:                  增加 P2 后:
  P3 (80x80) --> 中小目标      P2 (160x160) --> 极小目标 (新增)
  P4 (40x40) --> 中目标        P3 (80x80)  --> 小目标
  P5 (20x20) --> 大目标        P4 (40x40)  --> 中目标
                               P5 (20x20)  --> 大目标
```

| 配置 | mAP@0.5 | mAP@0.5:0.95 | 参数量增加 | 速度影响 |
|------|---------|-------------|-----------|----------|
| Baseline (P3-P5) | 18.5% | 10.2% | - | 基准 |
| + P2 Head | 22.1% | 13.5% | +15% | -20% FPS |
| + P2 + P6 | 23.0% | 14.1% | +30% | -35% FPS |

### 5.3 SAHI 切片推理

SAHI (Slicing Aided Hyper Inference) 无需重新训练即可提升小目标检测：

```
原始大图 (2000x1500)
    |
    v
切片为 640x640 子图 (overlap=0.2)
    |
    v
对每个子图独立运行 YOLO
    |
    v
坐标映射回原图 + 全局 NMS
```

| 配置 | mAP@0.5 | 推理时间 |
|------|---------|----------|
| YOLOv8-s @ 640 | 19.8% | 15ms |
| YOLOv8-s @ 640 + SAHI | 27.5% | 45ms |
| YOLOv8-s @ 1280 | 26.3% | 35ms |

SAHI 以约 3 倍推理时间换取接近 1280 分辨率的精度，且无需重新训练。

### 5.4 数据增强策略

| 策略 | mAP 提升 | 说明 |
|------|----------|------|
| Mosaic | +2.5% | 拼接 4 张图，增加上下文多样性 |
| MixUp | +1.0% | 混合两张图，增强泛化 |
| CopyPaste | +3.2% | 复制小目标到新位置，缓解小目标不足 |
| Mosaic + CopyPaste | +4.5% | 组合效果最佳 |
| 随机旋转 (+-15 deg) | +1.8% | 模拟不同飞行角度 |

**CopyPaste 策略**: 从训练集中采样包含小目标的 patch，粘贴到其他图片合理位置，增加小目标出现频率约 30-50%。

**Mosaic 注意事项**: Mosaic 将 4 张图拼接导致每张图缩小 4x，小目标更加模糊。建议训练后期关闭 Mosaic 或降低 mosaic 比例。

### 5.5 Anchor 尺寸优化

使用 K-means 聚类 VisDrone 训练集标注框尺寸，生成针对性 anchor，mAP 提升约 1.8%。推荐 anchor 偏小：`[4,8, 8,16, 16,32, ...]`。

### 5.6 其他技巧

| 技巧 | 效果 | 难度 |
|------|------|------|
| 多尺度训练 | +1.5% | 低 |
| AutoAnchor | +1.2% | 低 |
| CIoU/WIoU Loss | +0.8% | 低 |
| Focal Loss (类别不平衡) | +1.5% | 中 |
| COCO->VisDrone 预训练 | +2.0% | 低 |
| CA 注意力 (避免 CBAM) | +1.0% | 低 |
| NMS 阈值调优 (0.55-0.65) | +0.5% | 低 |

## 7. 经验总结

### 7.1 什么方法有效

**高确定性有效**:
1. **提升输入分辨率** -- 640 到 1280 带来 7-8 个百分点提升，实现简单
2. **增加 P2 检测头** -- 对极小目标 (<16px) 效果显著
3. **CopyPaste 数据增强** -- 直接增加小目标样本，mAP 提升 3-4 个百分点
4. **SAHI 切片推理** -- 无需重新训练即可大幅提升性能
5. **大规模预训练** -- COCO->VisDrone 迁移比从零训练提升 2-3 个百分点

**中等确定性有效**:
6. 密度图引导的自适应裁剪 (密集场景效果好)
7. Focal Loss / 类别加权 (对长尾类别有帮助)
8. 多尺度训练 (增强尺度鲁棒性)

### 7.2 什么方法效果有限

1. **单纯增加模型容量** -- YOLOv8-s 到 YOLOv8-l 仅提升 2-3%，推理时间翻倍
2. **复杂后处理** -- Soft-NMS、DIoU-NMS 提升有限 (<1%)
3. **过度数据增强** -- 过强色彩抖动、过度 MixUp 反而降低性能
4. **从零训练大模型** -- 6000 张训练集上易过拟合
5. **CBAM 注意力** -- 空间注意力全局池化破坏小目标位置信息，mAP 下降

### 7.3 实践方案建议

**入门方案** (快速 baseline):
```
YOLOv8-s + 输入分辨率 1280 + Mosaic + MixUp
预期 mAP@0.5: ~28-30%
```

**进阶方案** (追求精度):
```
YOLOv8-m + 分辨率 1280 + P2 检测头 + CopyPaste + CA 注意力 + Anchor 重聚类
预期 mAP@0.5: ~34-36%
```

**部署方案** (速度-精度平衡):
```
YOLOv8-s + 分辨率 640 + SAHI 切片推理
预期 mAP@0.5: ~27-28%, FPS: ~30
```

**竞赛方案** (追求 SOTA):
```
Co-DETR + Swin-L Backbone + 多尺度训练 + TTA + 集成学习
预期 mAP@0.5: ~38-40%
```

### 7.4 常见陷阱

| 陷阱 | 表现 | 解决方案 |
|------|------|----------|
| Mosaic 过度 | 小目标 AP 下降 | 后期关闭或降低比例 |
| 锚框不适配 | 大量 anchor 与 GT 不匹配 | 重新 k-means 聚类 |
| NMS 过强 | 密集区域漏检 | 降低 IoU 阈值 (0.55-0.65) |
| 评估指标误导 | mAP@0.5 与 mAP@0.5:0.95 差距大 | 关注 mAP@0.5:0.95 |
| 类别不均 | 少数类 AP 极低 | 类别加权、focal loss |
| 显存溢出 | 1280 输入 OOM | 梯度累积、减小 batch |

### 7.5 未解决的挑战

| 挑战 | 现状 | 研究方向 |
|------|------|----------|
| 实时高精度 | 高精度方法均 <15 FPS | 蒸馏、剪枝、量化 |
| 夜间场景 | 训练数据不足 | 红外融合、GAN 增强 |
| 极端遮挡 | mAP <15% | 可见部分推理、姿态估计辅助 |
| 类别不平衡 | 罕见类别 mAP <10% | 长尾学习、元学习 |
| 泛化到新场景 | 跨域性能下降显著 | 域适应、无监督微调 |
