# 数据增强策略 (Data Augmentation)

> **阅读时间**: ~10 分钟 | **前置知识**: 目标检测基本概念、YOLO 系列基础架构

数据增强 (Data Augmentation) 是深度学习中最经济有效的正则化手段之一。对于目标检测任务而言，合理的增强策略不仅能显著缓解过拟合，还能在不增加标注成本的前提下成倍扩充训练数据的多样性。YOLO 系列从 v4 开始大量引入创新性增强方法，成为其高精度的重要原因之一。

## 1. 为什么需要数据增强

### 1.1 防止过拟合

当训练数据量有限时，模型容易"记住"训练样本而非学习通用特征，导致在验证集和测试集上表现不佳。数据增强通过对训练样本施加随机变换，使每次 epoch 看到的图像都略有不同，从而有效扩大训练数据的"表观"规模。

### 1.2 提升泛化能力

真实场景中目标会以不同尺度、角度、光照条件出现。增强策略模拟了这些变化，使模型对各种实际条件更加鲁棒 (robust)。

### 1.3 数据增强的分类

```
数据增强
├── 像素级 (Pixel-level)
│   ├── 颜色抖动 (Color Jitter)
│   ├── 噪声注入 (Noise Injection)
│   └── 随机擦除 (Random Erasing)
├── 几何级 (Geometric)
│   ├── 翻转 (Flip)
│   ├── 旋转 (Rotation)
│   ├── 缩放 (Scale)
│   └── 仿射变换 (Affine Transform)
└── 多图混合 (Multi-image)
    ├── Mosaic
    ├── MixUp
    ├── CutMix
    └── Copy-Paste
```

> **注意**: 目标检测的数据增强必须同步变换标注框 (bounding box)，否则标注会与图像不对应，导致训练失败。

## 2. 几何变换 (Geometric Transformations)

几何变算是最基础也最常用的增强手段。下图展示了主要的几何变换类型：

```
原始图像           水平翻转          旋转 15°         随机缩放
+--------+       +--------+       +--------+       +--------+
|  [car] |       |  [car] |       | [car]  |       |        |
|   →    |  -->  |    ←   |  -->  |   ↗    |  -->  | [car]  |
|        |       |        |       |        |       | (放大)  |
+--------+       +--------+       +--------+       +--------+
```

### 2.1 随机翻转 (Random Flip)

| 变换类型 | 说明 | 典型概率 |
|---------|------|---------|
| 水平翻转 (Horizontal Flip) | 沿垂直轴镜像 | p = 0.5 |
| 垂直翻转 (Vertical Flip) | 沿水平轴镜像 | 通常不使用 (自然场景中上下颠倒较少) |

水平翻转是几乎所有检测 pipeline 的默认增强。对于对称性较低的类别 (如文字)，需谨慎使用。

### 2.2 随机旋转 (Random Rotation)

在给定角度范围内随机旋转图像和对应的 bounding box：

```
角度范围: [-degrees, +degrees]

  0° (原图)        +30°             -15°
+---------+     +---------+      +---------+
| ┌─────┐ |     | ┌──╮    |      |   ┌─────┐|
| │ obj │ |     | │  ╰─╮  |      |   │ obj │|
| └─────┘ |     | ╰──╮  │ |      |   └─────┘|
|         |     |    ╰──┘ |      |          |
+---------+     +---------+      +---------+
```

YOLOv8 默认 `degrees=0.0` (关闭旋转)。开启时建议范围 `[-10, 10]` 度，过大的旋转角会导致有效像素大量丢失。

### 2.3 随机缩放 (Random Scale)

在 `[1 - scale, 1 + scale]` 范围内随机缩放图像。缩放后超出边界的区域用灰色填充或直接裁剪。

YOLOv8 默认 `scale=0.5`，即缩放范围为 `[0.5, 1.5]`。

### 2.4 随机平移 (Random Translate)

在水平和垂直方向上随机平移图像一定比例。平移有助于模型学习处理部分遮挡 (partial occlusion) 的目标。

YOLOv8 默认 `translate=0.1`，即最大平移图像宽/高的 10%。

### 2.5 仿射变换 (Affine Transformation)

仿射变换是翻转、旋转、缩放、平移的统一数学表示，可通过一个 2x3 矩阵描述：

```
| x' |   | a₁₁  a₁₂  tₓ |   | x |
| y' | = | a₂₁  a₂₂  tᵧ | × | y |
                              | 1 |
```

部分高级增强库 (如 Albumentations) 支持更灵活的仿射变换组合。

## 3. 颜色空间增强 (Color Augmentation)

颜色增强在像素级别改变图像外观，模拟不同光照、相机设置下的成像效果。

### 3.1 HSV 空间调整

将图像从 RGB 转换到 HSV (Hue, Saturation, Value) 空间，分别对三个通道施加随机扰动：

```
HSV 三通道示意：

H (色调 - Hue)          S (饱和度 - Saturation)     V (明度 - Value)
┌──────────────────┐    ┌──────────────────┐    ┌──────────────────┐
│ 色彩种类的变化     │    │ 颜色鲜艳程度       │    │ 画面明暗程度       │
│ 红→橙→黄→绿→蓝→紫 │    │ 灰暗 ←→ 鲜艳      │    │ 暗   ←→   亮      │
└──────────────────┘    └──────────────────┘    └──────────────────┘
```

| 参数 | YOLOv8 默认值 | 说明 |
|------|-------------|------|
| `hsv_h` | 0.015 | 色调偏移范围 (归一化到 [0,1]) |
| `hsv_s` | 0.7 | 饱和度缩放范围 |
| `hsv_v` | 0.4 | 明度缩放范围 |

### 3.2 对比度与亮度

除了 HSV 调整外，还可以直接在 RGB 空间调整对比度 (contrast) 和亮度 (brightness)。常用的方式是乘以一个随机系数：

```
pixel_new = alpha * pixel_old + beta

alpha: 对比度系数, 范围 [1-contrast, 1+contrast]
beta:  亮度偏移, 范围 [-brightness, +brightness]
```

### 3.3 颜色增强的注意事项

- 颜色增强不改变 bounding box 的位置和大小
- 过强的颜色增强可能导致物体外观严重失真，建议保守使用
- 对于颜色是关键区分特征的类别 (如红绿灯)，色调调整需特别小心

## 4. Mosaic 增强

Mosaic 是 YOLOv4 (2020) 引入的标志性增强方法，至今仍是 YOLO 系列的核心增强策略之一。

### 4.1 原理

从训练集中随机选取 4 张图像，分别进行随机缩放和翻转后，拼接为一张大图：

```
Mosaic 拼接过程：

随机选 4 张图:        随机缩放翻转后:          最终拼接:
+----+  +----+      +------+------+      +------+------+
| A  |  | B  |  --> | A'   |  B'  |  --> | A'   |  B'  |
+----+  +----+      +------+------+      +------+------+
| C  |  | D  |      | C'   |  D'  |      | C'   |  D'  |
+----+  +----+      +------+------+      +------+------+

拼接中心点随机偏移，增加多样性
```

### 4.2 实现步骤

1. 随机选取 4 张训练图像及其标注
2. 随机确定拼接中心点 (cx, cy)
3. 对每张图进行随机缩放 (scale jitter)
4. 将 4 张图放置到对应象限
5. 调整 bounding box 坐标到新图像坐标系
6. 裁剪超出边界的 box

### 4.3 Mosaic 的优点

| 优点 | 说明 |
|------|------|
| **丰富上下文** | 4 张不同场景的图像拼接，增加背景多样性 |
| **小目标增强** | 缩放后小目标在拼接图中仍保留像素信息，相当于增加了小目标样本 |
| **Batch Normalization 统计** | 一张图包含 4 张图的信息，BN 统计更稳定 |
| **减少 GPU 显存需求** | 相当于一个 batch 包含 4 张图的信息，但只需处理一张拼接图 |

### 4.4 Mosaic 的缺点与改进

- **训练后期有害**: 拼接引入的边界伪影 (border artifacts) 在训练后期可能影响精度
- **YOLOv5 改进**: 训练最后 10 个 epoch 关闭 Mosaic
- **YOLOv8 改进**: 引入 Close Mosaic 策略，在最后 N 个 epoch 逐步降低 Mosaic 概率至 0

```
Mosaic 概率随训练进程变化 (YOLOv8):

概率
1.0 |████████████████████████░░░░░
    |                            ░░░░
0.5 |                                ░░░░
    |                                    ░░
0.0 |                                      ████
    +-----------------------------------------> epoch
    0                              close_mosaic   total
```

## 5. MixUp 增强

### 5.1 原理

MixUp 将两张图像按比例混合叠加，同时混合对应的标签：

```
I_mix = alpha * I_1 + (1 - alpha) * I_2
L_mix = alpha * L_1 + (1 - alpha) * L_2

其中 alpha ~ Beta(a, b), 通常 a = b = 1.2 或 a = b = 8.0
```

可视化效果：

```
图像 A (60%)         图像 B (40%)          MixUp 结果
+----------+       +----------+       +----------+
| ┌──┐     |       |     ┌──┐ |       | ┌──┐┌──┐ |
| │cat│     | ×0.6 + │ dog│ ×0.4 =   |半透明叠加  |
| └──┘     |       |     └──┘ |       |  两个物体  |
+----------+       +----------+       +----------+
```

### 5.2 在 YOLO 中的应用

- YOLOv5: 默认关闭，`mixup=0.0`
- YOLOv8: 默认 `mixup=0.0`，建议值 `0.0 ~ 0.15`
- 通常与 Mosaic 结合使用：先 Mosaic 拼接 4 张图，再与另一张 Mosaic 结果做 MixUp

### 5.3 MixUp 的优缺点

| 优点 | 缺点 |
|------|------|
| 提升模型校准性 (calibration) | 混合后物体边界模糊 |
| 增强正则化效果 | 标签变为软标签，可能影响检测精度 |
| 改善分类置信度 | 过强的混合会降低图像可辨识度 |

## 6. Copy-Paste 增强

### 6.1 原理

Copy-Paste 由 Google 在《Simple Copy-Paste is a Strong Data Augmentation Method for Instance Segmentation》(2021) 中提出。核心思想是：将一张图中的目标实例 (instance) 复制粘贴到另一张图的随机位置。

```
源图像 A                  目标图像 B              Copy-Paste 结果
+----------+            +----------+            +----------+
| ┌──┐     |            |          |            | ┌──┐     |
| │obj│     |  --复制--> |          | --粘贴-->  | │obj│[   ]|
| └──┘     |            |   [   ]  |            | └──┘[   ]|
+----------+            +----------+            +----------+
  源实例                   背景图                   合成结果
```

### 6.2 实现要点

1. 需要实例分割标注 (instance segmentation) 或至少精确的前景掩码
2. 复制时通常对实例边缘做高斯模糊处理，使粘贴更自然
3. 粘贴位置随机，但需避免与已有目标严重重叠
4. 可结合随机缩放粘贴实例，模拟不同距离

### 6.3 对小目标的效果

Copy-Paste 对小目标检测特别有效，原因：

- 可以将小目标从近距离图像复制到远距离场景，增加小目标样本
- 不依赖图像插值 (不像 Mosaic 缩放)，小目标像素信息不丢失
- 可控地增加特定类别的样本数量

## 7. CutMix 增强

### 7.1 原理

CutMix 从一张图像中裁剪一个矩形区域，粘贴到另一张图像的对应位置，标签按面积比例混合：

```
CutMix 过程：

图像 A                    图像 B                  CutMix 结果
+----------+            +----------+            +----------+
|          |            |          |            |    ┌───┐ |
|    A     |            |    B     |            | A  │ B │ |
|          |            |          |            |    └───┘ |
+----------+            +----------+            +----------+

标签: A 标签 * (1-面积比) + B 标签 * 面积比
```

### 7.2 与 MixUp 的区别

| 特性 | MixUp | CutMix |
|------|-------|--------|
| 混合方式 | 全图像素级加权叠加 | 局部区域替换 |
| 信息保留 | 两张图信息均被稀释 | 替换区域外信息完整 |
| 空间局部性 | 无 | 有，替换区域清晰 |
| 适用场景 | 分类为主 | 检测和分类均可 |

### 7.3 在检测中的应用

CutMix 在目标检测中使用较少，因为矩形替换可能切断目标实例，导致标注不准确。但在分类预训练阶段仍有价值。

## 8. Random Erasing / CutOut

### 8.1 Random Erasing

随机选择图像中的一个矩形区域，用随机值或均值填充，模拟遮挡 (occlusion)：

```
原始图像               Random Erasing
+----------+          +----------+
| ┌──────┐ |          | ┌──────┐ |
| │      │ |    -->   | │ ██░░ │ |   ██ = 被擦除区域
| │      │ |          | │ ██░░ │ |   随机填充值
| └──────┘ |          | └──────┘ |
+----------+          +----------+
```

### 8.2 CutOut

CutOut 是 Random Erasing 的简化版本，擦除区域固定用 0 (黑色) 填充。

### 8.3 在检测中的局限

- 随机擦除可能覆盖目标的关键特征区域
- 不同步处理 bounding box 标注，可能导致框与实际可见区域不匹配
- YOLOv8 未内置此策略，但可通过 Albumentations 等外部库实现

## 9. YOLOv8 默认增强策略

YOLOv8 (Ultralytics) 在 `default.yaml` 中定义了完整的增强参数。以下是完整列表及说明：

| 参数名 | 默认值 | 类型 | 说明 |
|--------|--------|------|------|
| `hsv_h` | 0.015 | float | HSV-色调增强范围 (fraction) |
| `hsv_s` | 0.7 | float | HSV-饱和度增强范围 (fraction) |
| `hsv_v` | 0.4 | float | HSV-明度增强范围 (fraction) |
| `degrees` | 0.0 | float | 旋转角度范围 (degrees) |
| `translate` | 0.1 | float | 平移范围 (fraction of image size) |
| `scale` | 0.5 | float | 缩放范围 (gain range = [1-scale, 1+scale]) |
| `shear` | 0.0 | float | 剪切角度 (degrees) |
| `perspective` | 0.0 | float | 透视变换系数 |
| `flipud` | 0.0 | float | 垂直翻转概率 |
| `fliplr` | 0.5 | float | 水平翻转概率 |
| `bgr` | 0.0 | float | BGR 通道随机翻转概率 |
| `mosaic` | 1.0 | float | Mosaic 增强概率 |
| `mixup` | 0.0 | float | MixUp 增强概率 |
| `copy_paste` | 0.0 | float | Copy-Paste 增强概率 (需实例分割标注) |
| `erasing` | 0.4 | float | Random Erasing 概率 (训练分类时) |
| `crop_fraction` | 1.0 | float | 图像裁剪比例 (分类任务) |
| `close_mosaic` | 10 | int | 最后 N 个 epoch 关闭 Mosaic |

### YOLOv8 默认增强配置 (YAML)

```yaml
# YOLOv8 默认增强参数 (简化版)
augment:
  hsv_h: 0.015        # 色调
  hsv_s: 0.7          # 饱和度
  hsv_v: 0.4          # 明度
  degrees: 0.0        # 旋转 (默认关闭)
  translate: 0.1      # 平移
  scale: 0.5          # 缩放
  shear: 0.0          # 剪切 (默认关闭)
  perspective: 0.0    # 透视 (默认关闭)
  flipud: 0.0         # 上下翻转 (默认关闭)
  fliplr: 0.5         # 左右翻转
  mosaic: 1.0         # Mosaic (默认开启)
  mixup: 0.0          # MixUp (默认关闭)
  copy_paste: 0.0     # Copy-Paste (默认关闭)
  close_mosaic: 10    # 最后10个epoch关闭Mosaic
```

### 训练策略建议

- **小数据集 (< 1000 张)**: 开启更多增强 (mixup=0.1, degrees=10, scale=0.8)
- **大数据集 (> 10000 张)**: 默认增强即可，可适当降低 Mosaic
- **小目标场景**: 开启 Copy-Paste，保持 Mosaic=1.0，close_mosaic=15~20
- **高精度需求**: 后期关闭所有增强 (fine-tune)，只保留基础翻转

## 10. 增强策略对比

### 10.1 综合对比表

| 增强方法 | 改变内容 | 是否改变 bbox | 计算开销 | 对小目标效果 | 引入版本 |
|---------|---------|--------------|---------|------------|---------|
| 水平翻转 | 像素 + bbox | 是 (镜像) | 极低 | 一般 | 通用 |
| 随机旋转 | 像素 + bbox | 是 (旋转) | 低 | 一般 | 通用 |
| 随机缩放 | 像素 + bbox | 是 (缩放) | 低 | 有效 | 通用 |
| 随机平移 | 像素 + bbox | 是 (平移) | 低 | 一般 | 通用 |
| HSV 调整 | 仅像素 | 否 | 极低 | 无直接效果 | 通用 |
| Mosaic | 拼接 4 张图 | 是 (重算) | 中 | **非常有效** | YOLOv4 |
| MixUp | 像素叠加 | 软标签 | 中 | 一般 | YOLOv5 |
| Copy-Paste | 实例粘贴 | 新增 bbox | 高 | **非常有效** | Google (2021) |
| CutMix | 区域替换 | 部分 bbox 丢失 | 低 | 一般 | CutMix (2019) |
| Random Erasing | 部分遮挡 | 否 | 极低 | 负面影响 | Random Erasing (2017) |

### 10.2 对 mAP 影响的经验估计

以下数据基于 COCO 数据集的典型实验结果，仅供参考：

| 增强组合 | 相对 mAP 变化 | 备注 |
|---------|-------------|------|
| 无增强 (baseline) | 基准 | 容易过拟合 |
| + 翻转 + 缩放 | +2~3% | 基础收益 |
| + Mosaic | +2~4% | 小目标提升显著 |
| + MixUp | +0.5~1% | 校准性改善 |
| + Copy-Paste | +1~3% (实例分割) | 需要分割标注 |
| 全部开启 (激进) | 可能 +5~8% | 需要调参 |

## 11. 小目标检测的增强建议

小目标 (COCO 定义: 面积 < 32x32 像素) 检测是目标检测的难题之一。合理的增强策略对小目标至关重要。

### 11.1 推荐策略

| 策略 | 推荐值 | 理由 |
|------|--------|------|
| Mosaic | 1.0 (全程开启) | 4 图拼接相当于降低单图分辨率，小目标相对变大 |
| close_mosaic | 15~20 | 最后阶段逐步关闭，避免边界伪影 |
| Copy-Paste | 0.3~0.5 | 直接增加小目标实例数量 |
| scale | 0.5~0.8 | 随机缩放模拟不同距离 |
| 图像尺寸 | 1280 (而非 640) | 更大输入尺寸保留更多小目标信息 |
| TTA (测试增强) | 开启多尺度测试 | 测试时使用 0.5x, 1.0x, 1.5x 等尺度 |

### 11.2 Mosaic 对小目标的原理

```
原始 640x640 图像:                Mosaic 后 640x640:
+-------------------+           +--------+--------+
|                   |           |  320x  |  320x  |
|  一张图中的小目标   |    -->    |  320   |  320   |
|  (可能只有几个像素) |           +--------+--------+
|                   |           |  320x  |  320x  |
+-------------------+           |  320   |  320   |
                                +--------+--------+
  小目标在大图中很小              小目标在拼接图中仍保留
  (如 8x8 像素)                 原始比例，但更多上下文
```

Mosaic 本身不放大小目标，但配合随机缩放，其中 1~3 张图会被放大，从而使这些图中的小目标获得更大的像素表示。

### 11.3 Copy-Paste 增强小目标流程

```
1. 从训练集随机选择源图像 (包含小目标)
2. 提取小目标实例及其掩码 (mask)
3. 随机缩放实例 (可放大以增加多样性)
4. 选择目标图像的随机位置粘贴
5. 更新标注: 新增 bounding box 和类别

源图 (远距离)          目标图 (近距离场景)
+----------+          +------------------+
|  小目标   |          |                  |
|  (8x8)   |  ----->  |    [粘贴的小目标]  |
+----------+          |                  |
                      +------------------+
```

### 11.4 避免的错误

| 错误做法 | 正确做法 |
|---------|---------|
| 训练后期才关闭 Mosaic | 使用 close_mosaic 渐进关闭 |
| 小数据集不开增强 | 小数据集更需要强增强 |
| 所有增强都开到最大 | 根据数据集大小和任务调参 |
| 忽略输入分辨率 | 小目标检测优先使用大分辨率 |
| 只关注 mAP | 关注小目标子集的 AP_S 指标 |

## 思考题

<details>
<summary>1. 为什么 YOLOv8 默认关闭 MixUp (`mixup=0.0`)，而 Mosaic 默认开启 (`mosaic=1.0`)？</summary>

Mosaic 通过拼接 4 张图扩充了上下文多样性，且对小目标特别有效，收益明确。MixUp 的像素级叠加会使两张图像都变得模糊，在检测任务中可能导致目标边界不清晰，影响定位精度。此外，MixUp 产生软标签 (soft label)，对回归任务的优化不如对分类任务直接。因此 YOLOv8 选择保守默认值，需要时手动开启。
</details>

<details>
<summary>2. 在训练的最后阶段关闭 Mosaic (`close_mosaic`) 的原理是什么？</summary>

Mosaic 拼接会在图像中引入人为的边界线 (border artifacts)，且拼接后的图像不完全符合真实场景的统计分布。训练初期使用 Mosaic 能有效增加数据多样性；但在训练后期，模型需要在"干净"的真实图像上 fine-tune 以获得精确的检测能力。逐步关闭 Mosaic 而非突然关闭，可以让模型平稳过渡，避免损失突增。
</details>

<details>
<summary>3. 如果你的数据集只有 500 张图像且包含大量小目标，你会如何设计增强策略？</summary>

建议策略：(1) Mosaic=1.0, close_mosaic=15; (2) Copy-Paste=0.4 (需实例分割标注); (3) fliplr=0.5, scale=0.5; (4) hsv_h=0.015, hsv_s=0.7, hsv_v=0.4; (5) 输入分辨率 1280; (6) 可选 mixup=0.1。同时建议结合 Albumentations 库增加更多变换。数据量极少时应开启较强的增强以防止过拟合，但需监控验证集表现。
</details>

<details>
<summary>4. Copy-Paste 增强为什么需要实例分割标注而非仅需 bounding box 标注？</summary>

因为 Copy-Paste 需要精确提取目标的前景像素。仅用 bounding box 会包含大量背景像素，粘贴后会带入源图的背景信息，产生不自然的拼接效果。实例分割的掩码 (mask) 能精确勾勒目标轮廓，使得粘贴后的结果更自然，且可以对边缘做羽化处理以提升融合质量。
</details>

<details>
<summary>5. 增强策略是否越多越好？请分析增强的"度"的问题。</summary>

并非越多越好。过强的增强会导致：(1) 训练图像严重失真，模型难以学习有意义的特征；(2) 训练收敛变慢甚至不收敛；(3) 标注与增强后图像不匹配 (如旋转导致目标部分出界)。增强的"度"应根据数据集大小、任务复杂度和模型容量来调整。通常的做法是：先用默认增强训练作为 baseline，再逐步添加或调参，通过验证集 mAP 选择最优配置。
</details>
