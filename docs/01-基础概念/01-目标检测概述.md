## 1. 目标检测定义与任务分类

### 1.1 什么是目标检测

目标检测的任务是：给定一张输入图像，**同时**完成以下两个子任务——

1. **定位(Localization)**：用边界框(Bounding Box)标出每个目标的位置
2. **分类(Classification)**：判断每个边界框内目标的类别

输出格式通常为：`(class, x, y, w, h, confidence)` 或 `(class, x1, y1, x2, y2, confidence)`

### 1.2 三大视觉任务对比

| 任务 | 输入 | 输出 | 粒度 | 典型应用 |
|------|------|------|------|----------|
| 图像分类(Classification) | 图像 | 类别标签 | 整张图像 | 图像搜索、医学诊断 |
| 目标检测(Detection) | 图像 | 类别 + 边界框 | 目标级别 | 自动驾驶、安防监控 |
| 语义分割(Segmentation) | 图像 | 逐像素类别掩码 | 像素级别 | 医学图像、遥感分析 |
| 实例分割(Instance Segmentation) | 图像 | 逐实例掩码 + 类别 | 像素+实例级别 | 自动驾驶、机器人抓取 |

**任务关系图**：

```
图像 ──→ [分类] ──→ "这是一只猫"
  │
  ├──→ [检测] ──→ "猫在 (120,80)-(350,400)"
  │
  ├──→ [语义分割] ──→ 每个像素属于哪个类别(不区分实例)
  │
  └──→ [实例分割] ──→ 每个实例的像素级掩码(区分不同猫)
```

### 1.3 评估指标

- **IoU(Intersection over Union)**：预测框与真实框的交集面积 / 并集面积

```
IoU = Area(预测框 ∩ 真实框) / Area(预测框 ∪ 真实框)

    ┌──────────┐
    │  真实框   │
    │    ┌─────┼────┐
    │    │/////│    │
    └────┼─────┘    │
         │  预测框  │
         └──────────┘
    阴影部分 = 交集
```

- **mAP(mean Average Precision)**：在不同 IoU 阈值下的平均精度
  - `mAP@0.5`：IoU 阈值 = 0.5 (PASCAL VOC 标准)
  - `mAP@0.5:0.95`：IoU 阈值从 0.5 到 0.95 步长 0.05 取均值 (COCO 标准)
- **FPS(Frames Per Second)**：每秒处理帧数，衡量推理速度

## 2. Two-Stage Detectors: 两阶段检测器

两阶段检测器的核心思想是：**先生成候选区域(Region Proposals)，再对候选区域进行分类和回归**。

### 2.1 演进路线

```
R-CNN (2014)
  │  Selective Search → CNN特征提取 → SVM分类
  │  问题: 每张图~2000个候选区域分别过CNN，极慢(~47s/张)
  ▼
Fast R-CNN (2015)
  │  整张图过CNN → RoI Pooling → 全连接层分类+回归
  │  改进: 共享特征计算，速度提升~25x
  │  瓶颈: Selective Search 仍是速度瓶颈
  ▼
Faster R-CNN (2015)
  │  RPN(Region Proposal Network)替代Selective Search
  │  端到端训练，速度~5 FPS
  │  架构: Backbone → RPN → RoI Pooling → Head
  ▼
Cascade R-CNN (2018)
     级联多个检测头，逐步提高IoU阈值
     解决单一IoU阈值的训练-推理不匹配问题
```

### 2.2 Faster R-CNN 架构详解

Faster R-CNN 是两阶段检测器的里程碑，其架构至今仍影响着许多检测器的设计。

**整体流程**：

```
输入图像
    │
    ▼
┌─────────────┐
│   Backbone   │  (VGG / ResNet / ...)
│  特征提取    │
└──────┬──────┘
       │ Feature Map
       ├──────────────────────┐
       ▼                      ▼
┌─────────────┐       ┌──────────────┐
│    RPN       │       │  RoI Pooling  │
│ 候选区域生成  │──────→│  特征对齐     │
└─────────────┘       └──────┬───────┘
                              │
                              ▼
                     ┌──────────────┐
                     │   Head       │
                     │ 分类 + 回归   │
                     └──────────────┘
```

**RPN(Region Proposal Network)**：
- 在特征图上滑动 3x3 卷积
- 每个位置生成 k 个锚框(Anchor Boxes)，通常 k=9 (3种尺度 x 3种比例)
- 输出 2k 个前景/背景分数 + 4k 个边界框偏移量
- 通过 NMS(Non-Maximum Suppression)过滤，保留 Top-N 候选框

### 2.3 Cascade R-CNN

Cascade R-CNN 通过级联多个检测头来逐步精化检测结果：

```
RoI → Head_1 (IoU=0.5) → 精化RoI → Head_2 (IoU=0.6) → 精化RoI → Head_3 (IoU=0.7)
```

每个阶段使用不同的 IoU 阈值训练，后一阶段的检测头处理前一阶段输出的更高质量的候选框，从而逐步提升检测精度。

## 3. One-Stage Detectors: 单阶段检测器

单阶段检测器直接在特征图上进行密集预测，省去了候选区域生成步骤，速度更快。

### 3.1 YOLO 系列演进

**YOLO(You Only Look Once)** 是最具代表性的单阶段检测器系列：

```
YOLOv1 (2016)
  │  将图像划分为 S×S 网格，每个网格预测 B 个边界框
  │  速度极快(45 FPS)，但小目标检测能力弱
  ▼
YOLOv2 / YOLO9000 (2017)
  │  引入 Batch Normalization、Anchor Boxes、多尺度训练
  │  检测精度大幅提升，可检测 9000+ 类别
  ▼
YOLOv3 (2018)
  │  Darknet-53 backbone、FPN 多尺度检测
  │  三个检测头分别负责小/中/大目标
  ▼
YOLOv4 (2020)
  │  CSPDarknet53 + SPP + PANet + SAM
  │  大量训练技巧(Mosaic数据增强、CIoU Loss等)
  ▼
YOLOv5 (2020)
  │  PyTorch原生实现，工程化程度高
  │  自适应锚框计算、Focus结构
  ▼
YOLOv7 (2022)
  │  E-ELAN 结构、模型重参数化
  │  在精度和速度之间取得新平衡
  ▼
YOLOv8 / YOLOv9 / YOLOv10 / YOLO11 (2023-2024)
     持续优化：解耦头、轻量化、C2f模块、PGI/GELAN等
```

### 3.2 SSD (Single Shot MultiBox Detector, 2016)

SSD 在多个不同分辨率的特征图上进行检测，天然支持多尺度目标：

```
输入图像 (300×300)
    │
    ▼
VGG-16 Backbone → 额外卷积层
    │
    ├─→ Conv4_3  (38×38) → 检测小目标
    ├─→ Conv7    (19×19) → 检测中目标
    ├─→ Conv8_2  (10×10) →
    ├─→ Conv9_2  ( 5× 5) → 检测大目标
    ├─→ Conv10_2 ( 3× 3) →
    └─→ Conv11_2 ( 1× 1) →
         │
         ▼
    NMS → 最终检测结果
```

### 3.3 RetinaNet (2017) 与 Focal Loss

RetinaNet 的核心贡献是提出了 **Focal Loss**，解决了单阶段检测器中正负样本极度不均衡的问题。

**Focal Loss 公式**：

```
FL(p_t) = -α_t * (1 - p_t)^γ * log(p_t)

其中:
  p_t = p      (当真实标签=1时)
  p_t = 1 - p  (当真实标签=0时)
  α_t  = 平衡因子(通常取0.25)
  γ    = 聚焦参数(通常取2.0)
```

**直觉理解**：
- 当模型对某样本预测置信度高时，`(1 - p_t)^γ` 趋近于 0，该样本的损失被大幅降低
- 模型将"注意力"集中在难分类的样本上
- γ=0 退化为标准交叉熵；γ=2 时，对 p_t=0.9 的样本，损失降低约 100 倍

## 4. Anchor-Based vs Anchor-Free

### 4.1 Anchor-Based 方法的局限

传统基于锚框的方法存在以下问题：
- **超参数敏感**：锚框的尺度、比例、数量需要精心设计
- **正负样本不均衡**：大量锚框与目标的 IoU 很低
- **计算冗余**：每个位置生成大量锚框，大部分会被丢弃

### 4.2 Anchor-Free 方法

Anchor-Free 方法直接预测目标的关键点或中心点，无需预定义锚框。

**FCOS (Fully Convolutional One-Stage Object Detection, 2019)**：

```
特征图上的每个位置 (x, y)
    │
    ├─→ 是否在某个 ground-truth 框内？ ──→ 否 → 负样本
    │
    └─→ 是 → 正样本，回归目标:
            l = x - x_min
            t = y - y_min
            r = x_max - x
            b = y_max - y
            (即该位置到边界框四条边的距离)
```

**CenterNet (2019)**：
- 将目标检测为边界框的中心点
- 在热力图(Heatmap)上预测中心点位置
- 额外回归目标的宽高和偏移量
- 无需 NMS 后处理

**CornerNet (2018)**：
- 检测每个目标边界框的左上角和右下角
- 通过 Embedding 向量配对属于同一目标的角点

### 4.3 对比总结

| 特性 | Anchor-Based | Anchor-Free |
|------|-------------|-------------|
| 锚框设计 | 需要预定义尺度和比例 | 不需要 |
| 正负样本分配 | 基于 IoU 匹配 | 基于空间位置或中心区域 |
| 超参数 | 较多(尺度、比例、数量) | 较少 |
| NMS 后处理 | 通常需要 | 部分方法可省略 |
| 代表方法 | Faster R-CNN, RetinaNet, SSD | FCOS, CenterNet, CornerNet |
| 检测精度 | 成熟稳定，高精度 | 近年追平甚至超越 |
| 计算效率 | 锚框生成有额外开销 | 通常更简洁高效 |

## 5. DETR 与端到端检测器

### 5.1 DETR (Detection Transformer, 2020)

DETR 将目标检测视为一个**集合预测**问题，彻底摒弃了锚框、NMS 等手工设计组件。

**DETR 核心架构**：

```
输入图像
    │
    ▼
┌─────────────┐
│   CNN Backbone │  (ResNet)
└──────┬──────┘
       │ 展平特征序列
       ▼
┌─────────────┐
│ Transformer  │
│   Encoder    │  (6层, 全局自注意力)
└──────┬──────┘
       ▼
┌─────────────┐
│ Transformer  │
│   Decoder    │  (6层, N个Object Query)
└──────┬──────┘
       │ N个预测
       ▼
┌─────────────┐
│  FFN Heads   │
│ 分类 + 边界框 │
└─────────────┘
```

**关键创新**：
- **Object Queries**：N 个可学习的查询向量(通常 N=100)，每个查询负责预测一个目标或"无目标"
- **匈牙利匹配(Hungarian Matching)**：训练时通过二分图匹配将预测与真实标签一一对应
- **无需 NMS**：集合预测天然避免重复检测

### 5.2 DETR 的局限与改进

| 问题 | 改进方法 |
|------|----------|
| 训练收敛慢(500 epochs) | Deformable DETR: 可变形注意力，收敛速度提升 10x |
| 小目标检测效果差 | 多尺度 Deformable Attention |
| 计算复杂度高(O(n^2)) | Sparse DETR, Efficient DETR |
| 固定数量查询 | DN-DETR: 去噪训练，加速收敛 |

### 5.3 端到端检测的意义

```
传统流程:
  Backbone → 密集预测 → NMS → 最终结果
                ↑               ↑
            手工设计         后处理(不可微)

端到端流程:
  输入 ──→ Transformer ──→ 直接输出结果
                    ↑
              全部可学习
```

端到端检测器使得所有组件都可以通过梯度下降联合优化，避免了后处理带来的性能瓶颈和超参数调优。

## 6. 各类方法综合对比

### 6.1 性能与速度对比

| 方法类别 | 代表方法 | mAP@0.5:0.95 (COCO) | 推理速度(FPS) | Backbone | 特点 |
|---------|---------|---------------------|--------------|----------|------|
| Two-Stage | Faster R-CNN | ~37 | ~15 | ResNet-50 | 经典稳定，精度高 |
| Two-Stage | Cascade R-CNN | ~42 | ~12 | ResNet-101 | 级联精化，高精度 |
| One-Stage | SSD300 | ~25 | ~60 | VGG-16 | 速度快，精度一般 |
| One-Stage | RetinaNet | ~40 | ~15 | ResNet-101+FPN | Focal Loss，均衡 |
| One-Stage | YOLOv5-L | ~52 | ~100 | CSPDarknet | 工程化优秀 |
| One-Stage | YOLOv8-L | ~53 | ~100 | C2f+Darknet | 当前主流 |
| Anchor-Free | FCOS | ~42 | ~25 | ResNet-101 | 简洁优雅 |
| Anchor-Free | CenterNet | ~42 | ~30 | Hourglass | 无需NMS |
| End-to-End | DETR | ~43 | ~8 | ResNet-50 | 端到端，慢收敛 |
| End-to-End | Deformable DETR | ~46 | ~15 | ResNet-50 | 快收敛，多尺度 |

> 注: 以上数值为近似参考值，实际性能取决于实现细节、训练策略和硬件环境。

### 6.2 优缺点对比

| 方法 | 优点 | 缺点 |
|------|------|------|
| Two-Stage | 精度高、小目标检测好、研究成熟 | 速度较慢、结构复杂 |
| One-Stage YOLO | 速度快、工程化好、部署友好 | 密集小目标仍有挑战 |
| One-Stage RetinaNet | 正负样本均衡、精度接近两阶段 | 速度与两阶段相近 |
| Anchor-Free | 设计简洁、超参数少、无锚框开销 | 部分场景精度略低 |
| DETR 类 | 端到端、全局推理、无需NMS | 训练慢、计算量大、小目标弱 |

## 7. 检测头设计: Shared Head vs Decoupled Head

检测头(Head)是连接特征提取与最终预测的关键组件。

### 7.1 Shared Head (耦合头)

分类和回归共享同一个卷积分支：

```
特征图 ──→ 3×3 Conv ──→ 分类分支(1×1 Conv)
                    └──→ 回归分支(1×1 Conv)
```

- 优点：参数少、速度快
- 缺点：分类和回归任务存在梯度冲突
- 代表：Faster R-CNN 原始设计、SSD

### 7.2 Decoupled Head (解耦头)

分类和回归各自拥有独立的分支：

```
特征图 ──→ 3×3 Conv ──→ 3×3 Conv ──→ 1×1 Conv → 分类输出
                    └──→ 3×3 Conv ──→ 1×1 Conv → 回归输出
```

- 优点：分类和回归互不干扰，精度更高
- 缺点：参数量和计算量增加
- 代表：YOLOv8、FCOS、RetinaNet

### 7.3 设计选择建议

| 场景 | 推荐方案 |
|------|---------|
| 实时检测(速度优先) | Shared Head 或轻量级 Decoupled Head |
| 高精度检测 | Decoupled Head |
| 部署资源受限 | Shared Head + 重参数化 |

## 8. 特征提取: FPN, PANet, BiFPN

多尺度特征融合是检测不同大小目标的关键技术。

### 8.1 FPN (Feature Pyramid Network, 2017)

**自顶向下**的特征融合，将高层语义信息传递到低层：

```
C5 (1/32) ──→ 1×1 Conv ──→ P5
                │ 2x上采样
C4 (1/16) ──→ 1×1 Conv ──→ (+) ──→ P4
                │ 2x上采样
C3 (1/8)  ──→ 1×1 Conv ──→ (+) ──→ P3
                │ 2x上采样
C2 (1/4)  ──→ 1×1 Conv ──→ (+) ──→ P2

(+) 表示逐元素相加
```

- 优点：有效融合多尺度语义信息
- 局限：信息流是单向的(自顶向下)

### 8.2 PANet (Path Aggregation Network, 2018)

在 FPN 基础上增加了一条**自底向上**的路径：

```
自顶向下路径 (FPN):
  P5 → P4 → P3 → P2

自底向上路径 (新增):
  P2 → P3 → P4 → P5

最终特征: {P2, P3, P4, P5} 同时包含高层语义和底层细节
```

- 双向信息流动，增强了底层特征的语义信息
- YOLOv3/v4/v5 均采用此结构

### 8.3 BiFPN (Bidirectional Feature Pyramid Network, 2020)

EfficientDet 提出的加权双向特征金字塔网络：

```
P7_in  ────────────────────────────────────────────→ P7_out
  │ ↑                                                   ↑
P6_in  ──────────────────────────────────→ P6_td ──────→ P6_out
  │ ↑                                        ↑            ↑
P5_in  ──────────────────────→ P5_td ──────→ P5_out ─────┘
  │ ↑                                ↑
P4_in  ────────────→ P4_td ──────→ P4_out
  │ ↑                        ↑
P3_in  ──────→ P3_td ──────→ P3_out
  │ ↑                ↑
P2_in  ────→ P2_out ─┘

  ↓: 自顶向下融合
  ↑: 自底向上融合
```

**关键改进**：
- **加权特征融合**：为不同层的特征赋予可学习的权重，而非简单相加
- **去除单输入边节点**：如果某节点只有一个输入(无特征融合)，则跳过以减少计算
- **重复多次**：堆叠多个 BiFPN Block 以充分融合

### 8.4 三种结构对比

| 特性 | FPN | PANet | BiFPN |
|------|-----|-------|-------|
| 信息流方向 | 自顶向下 | 双向(先下后上) | 加权双向 |
| 融合方式 | 逐元素相加 | 逐元素相加 | 加权求和 |
| 融合次数 | 1次 | 2次 | 多次可堆叠 |
| 计算开销 | 低 | 中 | 中高 |
| 精度 | 基线 | 优于FPN | 优于PANet |
| 代表模型 | Faster R-CNN-FPN | YOLOv3/v4/v5 | EfficientDet |

## 思考题

**1. 为什么 YOLO 系列在工程部署中比 Faster R-CNN 更受欢迎？**

<details>
<summary>参考答案</summary>

YOLO 是单阶段检测器，推理流程简单直接：输入图像经过一次前向传播即可获得检测结果，无需生成候选区域和 RoI Pooling 等中间步骤。这使得 YOLO 在以下方面具有优势：
1. **推理速度快**：单次前向传播，FPS 远高于两阶段方法
2. **部署简单**：模型结构规整，易于 TensorRT / ONNX 等框架优化
3. **批处理友好**：密集预测模式对 batch inference 更高效
4. **工程生态完善**：Ultralytics 等团队提供了成熟的训练和部署工具链

而 Faster R-CNN 的 RPN + RoI Align + 双分支结构使其在部署优化时面临更多挑战。
</details>

**2. DETR 相比传统检测器的核心优势是什么？为什么它没有完全取代传统方法？**

<details>
<summary>参考答案</summary>

**核心优势**：
1. 端到端训练，无需 NMS 和锚框设计等手工组件
2. Transformer 的全局自注意力机制能捕获远距离依赖关系
3. 集合预测的思想优雅简洁，避免了重复检测

**未完全取代的原因**：
1. 训练收敛极慢(原始 DETR 需要 500 epochs，而 Faster R-CNN 约 36 epochs)
2. 小目标检测效果不如传统 FPN-based 方法
3. Transformer 的二次方计算复杂度限制了高分辨率特征图的使用
4. 对硬件算力要求较高，不适合边缘设备部署
5. 工业界追求稳定可预测的性能，传统方法经过多年优化更可靠
</details>

**3. FPN、PANet、BiFPN 三种特征融合方式各自适用于什么场景？**

<details>
<summary>参考答案</summary>

- **FPN**：适合计算资源有限、对速度要求高的场景。自顶向下的单次融合已经能有效提升小目标检测能力，是大多数检测器的基线选择。
- **PANet**：适合需要同时关注大目标和小目标的场景。双向路径确保底层特征也包含足够的语义信息，YOLO 系列广泛采用。
- **BiFPN**：适合追求最高精度且可以接受一定计算开销的场景。加权融合和多次堆叠能更充分地利用多尺度特征，EfficientDet 证明了其在精度-效率权衡上的优势。

实际选择时需综合考虑模型大小、推理延迟和精度需求。
</details>

**4. Anchor-Free 方法相比 Anchor-Based 方法有哪些本质优势？是否存在 Anchor-Free 方法无法处理的场景？**

<details>
<summary>参考答案</summary>

**本质优势**：
1. 消除了锚框设计的超参数调优负担
2. 减少了正负样本分配的复杂性
3. 降低了内存占用(无需存储大量锚框)
4. 更简洁的模型设计，便于理解和改进

**潜在局限**：
1. 对于极端长宽比的目标，基于中心点的 Anchor-Free 方法可能不如精心设计锚框比例的 Anchor-Based 方法
2. 密集排列的目标场景下，中心点可能重叠，需要额外的分组策略
3. 某些 Anchor-Free 方法对边界框回归的监督信号不如 Anchor-Based 方法直接

但总体而言，随着 FCOS、CenterNet 等方法的持续改进，Anchor-Free 方法在大多数场景下已能达到或超越 Anchor-Based 方法的性能。
</details>
