# 我们的实验总结

> 本文档记录基于 VisDrone 数据集、RTX 4060 Laptop (8GB) 平台的五轮实验全过程。
> 从基线模型到 SAHI 推理，从架构改进到失败教训，完整呈现实验思路和结论。

---

## 1. 实验背景

### 1.1 目标

在 VisDrone 数据集上进行**小目标检测**，探索提升 mAP 的有效方法。

### 1.2 硬件环境

```
GPU:  NVIDIA RTX 4060 Laptop
VRAM: 8 GB
RAM:  16 GB
```

### 1.3 关键约束

- VRAM 有限，batch size 受限 (通常 8~16)
- VisDrone 图片分辨率高 (约 2000x1500)，小目标占比大
- 类别分布不均衡 (car 远多于 tricycle)

### 1.4 评估指标

主要关注 **mAP@0.5**（IoU=0.5 时的 mean Average Precision），这是目标检测中最常用的指标。

---

## 2. 五轮实验总览

| 轮次 | 方法 | mAP@0.5 | 相对基线 | 关键变化 |
|------|------|---------|---------|---------|
| Round 1 | YOLOv8n 基线 | 0.2979 | - | 标准配置 |
| Round 2 | 改进增强策略 | 0.4258 | +42.9% | 数据增强优化 |
| Round 3 | SAHI 推理 | 0.4903 | +64.6% | 切片推理 |
| Round 4 | 零训练优化 | - | - | 后处理调优 |
| Round 5 | P2+CBAM | 0.2913 | -2.2% | 架构修改 (失败) |

---

## 3. Round 1: 基线实验

### 3.1 配置

```python
from ultralytics import YOLO

model = YOLO("yolov8n.pt")
model.train(
    data="visdrone.yaml",
    epochs=100,
    imgsz=640,
    batch=16,
    device=0,
    # 默认数据增强
)
```

### 3.2 结果

```
mAP@0.5:    0.2979
mAP@0.5:0.95: 0.1612
Precision:  0.4521
Recall:     0.3156
```

### 3.3 分析

- mAP@0.5 仅 0.2979，远低于 COCO 上的表现（YOLOv8n 在 COCO 上约 37% mAP@0.5:0.95）。
- **小目标检测困难：** VisDrone 中大量目标尺寸 < 32x32 像素。
- **类别不均衡：** car 类 AP 较高，tricycle、awning-tricycle 类 AP 很低。

---

## 4. Round 2: 数据增强优化

### 4.1 改进策略

```python
model.train(
    data="visdrone.yaml",
    epochs=100,
    imgsz=640,
    batch=16,
    device=0,
    # 增强参数调整
    hsv_h=0.015,      # 色调抖动
    hsv_s=0.7,        # 饱和度抖动
    hsv_v=0.4,        # 亮度抖动
    degrees=10.0,     # 旋转角度
    translate=0.2,    # 平移范围
    scale=0.9,        # 缩放范围 (重要: 增加小目标多样性)
    fliplr=0.5,       # 水平翻转
    mosaic=1.0,       # Mosaic 增强
    mixup=0.15,       # MixUp 增强 (新增)
    copy_paste=0.1,   # Copy-Paste 增强 (新增)
)
```

### 4.2 结果

```
mAP@0.5:    0.4258  (+42.9% vs Round 1)
mAP@0.5:0.95: 0.2315
Precision:  0.5102
Recall:     0.4021
```

### 4.3 关键发现

- **数据增强是最有效的手段！** 仅调整增强参数就提升了 42.9%。
- `mixup` 和 `copy_paste` 对小目标特别有效。
- `scale` 范围扩大（0.9 vs 默认 0.5）增加了模型对不同尺度的鲁棒性。

---

## 5. Round 3: SAHI 推理

### 5.1 方法

使用 SAHI（Slicing Aided Hyper Inference）将大图切分为多个 640x640 的小块，分别推理后融合结果。

```python
from sahi import AutoDetectionModel
from sahi.predict import get_sliced_prediction

detection_model = AutoDetectionModel.from_pretrained(
    model_type="ultralytics",
    model_path="runs/detect/round2/weights/best.pt",
    confidence_threshold=0.25,
    device="cuda:0",
)

result = get_sliced_prediction(
    image="test_image.jpg",
    detection_model=detection_model,
    slice_height=640,
    slice_width=640,
    overlap_height_ratio=0.2,
    overlap_width_ratio=0.2,
)
```

### 5.2 结果

```
mAP@0.5:    0.4903  (+15.2% vs Round 2, +64.6% vs Round 1)
mAP@0.5:0.95: 0.2687
Precision:  0.4856
Recall:     0.4712
```

### 5.3 关键发现

- **SAHI 效果显著！** 相比 Round 2 又提升了 15.2%。
- **原因分析：** 原图约 2000x1500，resize 到 640x640 后小目标信息大量丢失。SAHI 保持了原始分辨率的信息。
- **Recall 提升明显：** 从 0.4021 提升到 0.4712，说明 SAHI 找到了更多之前漏检的小目标。

---

## 6. Round 4: 零训练优化

### 6.1 方法

不重新训练模型，仅调整推理参数：

```python
# 调整置信度阈值
results = model.predict(
    source="test_images/",
    conf=0.15,        # 降低置信度阈值 (从 0.25 到 0.15)
    iou=0.5,          # 调整 NMS IoU 阈值
    max_det=500,      # 增加最大检测数
    agnostic_nms=True, # 类别无关 NMS
)
```

### 6.2 结果

- 降低 conf 阈值提升了 Recall，但 Precision 下降。
- 调整 iou 阈值对密集场景有帮助。
- **结论：** 零训练优化有小幅提升，但不如 SAHI 和数据增强显著。

---

## 7. Round 5: P2 + CBAM (失败)

### 7.1 动机

- **P2 层：** 增加 P2 detection head (160x160 分辨率)，提升小目标检测能力。
- **CBAM：** Convolutional Block Attention Module，通道 + 空间注意力机制。

### 7.2 配置

```yaml
# yolov8n-p2-cbam.yaml
backbone:
  # ... 标准 backbone ...
  - [-1, 1, CBAM, [256]]  # 在 P3 层后添加 CBAM

head:
  - [-1, 1, Conv, [256, 1, 1]]  # P2 detection head
  # ... 其余 head 层 ...
```

### 7.3 结果

```
mAP@0.5:    0.2913  (-2.2% vs Round 1 基线!)
mAP@0.5:0.95: 0.1578
Precision:  0.4201
Recall:     0.2834
```

### 7.4 失败原因分析

**CBAM 对小目标有害：**

```
CBAM 的空间注意力机制:
+-------------------+
| 0.1  0.8  0.3    |
| 0.2  0.9  0.4    |  注意力图倾向于关注
| 0.1  0.7  0.2    |  "显著"的区域
+-------------------+

小目标的问题:
- 小目标本身"不显著"
- CBAM 可能抑制了小目标的特征响应
- 注意力机制需要足够的上下文才能工作
```

**P2 层需要更大输入分辨率：**

```
P2 分辨率 = 输入尺寸 / 4
  640 / 4 = 160x160  (P2 在 640 输入下)
  1024 / 4 = 256x256 (P2 在 1024 输入下)

640 输入时, P2 层的特征图已经很大
  -> 计算量大幅增加
  -> 但小目标在 640 resize 后可能只有 2~3 像素
  -> P2 层也检测不到
```

## 8. 关键发现总结

### 8.1 发现 1: CBAM 对小目标有害

```
传统认知:          实验发现:
注意力机制         CBAM 的空间注意力
-> 聚焦重要区域    -> 倾向于关注"显著"区域
-> 提升检测        -> 小目标本身不显著
-> 对小目标有益    -> 反而抑制小目标特征
                   -> mAP 下降
```

**教训：** 不能盲目添加注意力模块，需要考虑目标尺度特性。

### 8.2 发现 2: P2 层需要 1024+ 输入

```
P2 detection head:
  - 增加计算量 (约 4x)
  - 在 640 输入下，小目标已因 resize 丢失信息
  - 需要 1024 或更大输入才能发挥作用
  - 但 1024 输入在 8GB VRAM 上 batch size 仅 4~8
  - 训练效率大幅下降
```

**教训：** P2 层不是万能的，需要配合大输入分辨率。

### 8.3 发现 3: SAHI 是最有效的方法

```
方法效果对比:
  数据增强:  +42.9%  (零成本, 训练时优化)
  SAHI:      +15.2%  (推理时优化, 无需重训练)
  P2+CBAM:   -2.2%   (架构修改, 反而有害)

SAHI 的优势:
  - 保持原始分辨率信息
  - 无需修改模型
  - 可与任何检测模型配合
  - 推理速度可接受 (切片并行)
```

### 8.4 发现 4: 数据增强比架构更重要

```
投入产出比:
  数据增强参数调整:  30 分钟 -> +42.9% mAP
  SAHI 推理设置:     1 小时  -> +15.2% mAP
  P2+CBAM 架构修改:  2 天    -> -2.2% mAP (失败)

结论: 在资源有限时, 优先优化数据和训练策略
```

## 9. 经验教训总结表

| 经验教训 | 详细说明 | 影响程度 |
|---------|---------|---------|
| 数据增强优先 | 调整增强参数比修改架构更有效 | 极高 |
| SAHI 是小目标利器 | 切片推理保持分辨率信息 | 高 |
| CBAM 不适合小目标 | 空间注意力会抑制不显著的目标 | 高 (负面) |
| P2 层需要大输入 | 640 输入下 P2 层无效 | 中 |
| 零训练优化有上限 | 调参能提升，但不如结构化改进 | 中 |
| 先跑基线再改进 | 没有基线就无法量化改进效果 | 极高 |
| 记录每次实验 | 实验日志是复现和总结的基础 | 极高 |
| VRAM 是硬约束 | 8GB 限制了 batch size 和输入尺寸 | 高 |
| 类别不均衡需处理 | 少数类别的 AP 极低，拉低整体 mAP | 中 |
| 验证集要固定 | 每次评估使用相同的验证集 | 极高 |

## 10. 后续改进方向

### 10.1 短期可尝试

- [ ] 更强的数据增强策略（如 Albumentations 自定义）
- [ ] 不同 IoU 损失函数（SIoU, WIoU v3）
- [ ] 类别平衡采样
- [ ] 更大模型（YOLOv8s/m）配合 SAHI

### 10.2 中期探索

- [ ] 自适应锚框（针对 VisDrone 小目标重新聚类）
- [ ] 多尺度训练（随机输入尺寸 480~1024）
- [ ] 预训练权重微调（在更大数据集上预训练）
- [ ] 集成学习（多模型 + WBF 融合）

### 10.3 长期方向

- [ ] Transformer-based 检测器（RT-DETR）
- [ ] 自监督预训练
- [ ] 自动数据增强搜索（AutoAugment）
- [ ] 知识蒸馏（大模型指导小模型）
