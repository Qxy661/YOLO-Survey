## 1. Cross Entropy Loss (交叉熵损失)

交叉熵衡量两个概率分布之间的差异。对于真实标签 $y$ 和预测概率 $p$：

**二分类** (Binary)：

```
CE(y, p) = -[y * log(p) + (1 - y) * log(1 - p)]
```

- $y=1$ 时：损失 = $-\log(p)$，要求 $p \to 1$
- $y=0$ 时：损失 = $-\log(1-p)$，要求 $p \to 0$

**多分类** (Multi-class)，使用 Softmax + Cross Entropy：

```
p_i = exp(z_i) / Σ_j exp(z_j)
CE = - Σ_{i=1}^{K} y_i * log(p_i) = -log(p_target)
```

| 特性 | 说明 |
|------|------|
| 梯度行为 | 对易分类样本也会产生较大的梯度 |
| 样本均衡 | 对正负样本一视同仁，无法处理不平衡 |
| 适用场景 | 类别均衡的分类任务（图像分类） |
| 局限 | 在密集检测中被大量简单负样本淹没 |

---

## 2. Binary Cross Entropy (BCE，二元交叉熵)

BCE 是 CE 在二分类场景的特例，也称 **Log Loss**。在目标检测中，一个目标可能同时属于多个类别（如 "人" 和 "骑手"），每个类别独立做二分类，使用 **Sigmoid** 而非 Softmax：

```
σ(z) = 1 / (1 + exp(-z))

BCE_total = - Σ_{c=1}^{K} [y_c * log(σ(z_c)) + (1 - y_c) * log(1 - σ(z_c))]
```

**YOLOv8 中的 BCE**：YOLOv8 在分类分支直接使用 BCE Loss，配合 Anchor-Free + Decoupled Head 架构：

```python
# YOLOv8 分类损失（简化）
loss = F.binary_cross_entropy_with_logits(
    pred_scores, target_scores, reduction='mean'
)
```

设计原因：支持多标签场景、配合 TaskAlignedAssigner 动态分配正负样本、梯度行为稳定。

> **数值稳定性**：PyTorch 的 `F.binary_cross_entropy_with_logits` 内部使用 log-sum-exp 技巧避免 $\log(\sigma(z))$ 的数值溢出。

## 3. Focal Loss (焦点损失，2017)

### 3.1 动机：类别不平衡

目标检测中正负样本比例可达 **1:1000**。大量简单负样本的梯度累积后主导总损失：

| 样本类型 | 数量 | 特点 | 对训练的影响 |
|----------|------|------|-------------|
| 简单负样本 | 极多 | 背景区域，置信度高 | 梯度累积后主导 loss |
| 困难负样本 | 少 | 目标边缘、遮挡 | 有价值但被淹没 |
| 正样本 | 少 | 真实目标区域 | 贡献不足 |

### 3.2 核心公式

**Focal Loss** 由 Lin et al. 在 **RetinaNet** (CVPR 2017) 中提出：

```
定义 p_t:
    p_t = p          当 y = 1（正样本）
    p_t = 1 - p      当 y = 0（负样本）

Focal Loss:
    FL(p_t) = -α_t * (1 - p_t)^γ * log(p_t)

展开为二分类形式:
    FL(y, p) = -α * y * (1-p)^γ * log(p)
               - (1-α) * (1-y) * p^γ * log(1-p)
```

### 3.3 调制因子 $(1-p_t)^\gamma$

核心创新在于降低简单样本的权重，让模型聚焦于困难样本：

```
                    样本分类      p_t   (1-p_t)^γ (γ=2)    调制效果
简单正样本 (Easy Positive)    p=0.95   0.95    0.0025      大幅降低权重
困难正样本 (Hard Positive)    p=0.6    0.6     0.16        权重保持较高
简单负样本 (Easy Negative)    p=0.05   0.95    0.0025      大幅降低权重
困难负样本 (Hard Negative)    p=0.4    0.6     0.16        权重保持较高
```

- $p_t \to 1$（分类正确）：$(1-p_t)^\gamma \to 0$，loss 被大幅缩小
- $p_t \to 0$（分类错误）：$(1-p_t)^\gamma \to 1$，loss 几乎不受影响

### 3.4 平衡因子 $\alpha_t$

$\alpha_t$ 平衡正负样本重要性：正样本权重 $\alpha$，负样本权重 $1-\alpha$。

### 3.5 实验结论

| $\gamma$ | $\alpha$ | AP (COCO) | 说明 |
|----------|----------|-----------|------|
| 0 | 0.5 | 31.1 | 等价于标准 CE |
| 1.0 | 0.5 | 32.7 | 明显提升 |
| **2.0** | **0.25** | **33.9** | **最优配置** |
| 5.0 | 0.25 | 32.7 | 过度抑制，性能下降 |

**最优超参数**: $\gamma=2$, $\alpha=0.25$。直觉上 $\alpha=0.25$ 看似降低正样本权重，但 Focal 机制大幅压低了简单负样本的权重，使得正样本的有效贡献仍然突出。

## 4. VariFocal Loss (VFL，变分焦点损失，2021)

### 4.1 动机

Focal Loss 对正负样本的处理是**对称的**。VariFocal (Zhang et al., 2021) 认为应**不对称处理**：正样本应考虑质量（IoU），负样本只需降低简单样本权重。

### 4.2 核心公式

```
定义: p — 预测分类分数, q — 目标质量分数 (IoU)

正样本 (q > 0):  VFL(p, q) = -q * [q * log(p) + (1-q) * log(1-p)]
负样本 (q = 0):  VFL(p, q) = -α * p^γ * log(1-p)
```

**关键区别**：
- **正样本**：用质量分数 $q$（IoU）加权，高质量样本贡献更大
- **负样本**：Focal 形式 $p^\gamma$ 降低简单负样本权重
- 实现了 **IoU-aware 分类**：分类标签 = IoU 值，推理时无需额外乘 IoU

### 4.3 效果

在密集检测器上提升显著：FCOS + VFL 相比 FCOS + Focal Loss，AP 从 38.7 提升至 41.6（COCO），尤其在高 IoU 阈值下表现更好。

## 5. Quality Focal Loss (QFL，质量焦点损失)

### 5.1 核心思想

QFL 由 **Dynamic FCOS** 提出，将分类标签从离散 $\{0, 1\}$ 扩展为**连续质量分数** $\hat{y} \in [0, 1]$（如 IoU）：

```
QFL(σ, ŷ) = -|ŷ - σ|^β * log(σ_t)

其中:
  σ — Sigmoid 输出
  σ_t = σ       当 ŷ >= 0.5
  σ_t = 1 - σ   当 ŷ < 0.5
  β — 聚焦参数（推荐 β=2）
```

### 5.2 与 Focal Loss 的关系

当标签为离散值时 QFL 退化为标准 Focal Loss：$\hat{y}=1$ 时等价于正样本项，$\hat{y}=0$ 时等价于负样本项。

**Dynamic FCOS** 利用 QFL 实现动态正负样本分配：分类分数 = $P_{cls} \times P_{IoU}$，高质量预测获得更多正样本分配。

## 6. 分类损失函数对比

| 损失函数 | 公式 | 核心特点 | 正负样本处理 | 典型应用 |
|---------|------|---------|-------------|---------|
| **Cross Entropy** | $-y\log(p)-(1-y)\log(1-p)$ | 基础分类损失 | 对称，无区分 | 图像分类 |
| **BCE** | $\sum_c -[y_c\log\sigma_c+(1-y_c)\log(1-\sigma_c)]$ | 多标签独立分类 | 各类别独立 | YOLOv8 |
| **Focal Loss** | $-\alpha_t(1-p_t)^\gamma\log(p_t)$ | 降低简单样本权重 | 对称聚焦 | RetinaNet, YOLOv3 |
| **VariFocal** | 正:$-q[q\log p+(1-q)\log(1-p)]$; 负:$-\alpha p^\gamma\log(1-p)$ | 不对称 + IoU-aware | 不对称 | FCOS+, ATSS+ |
| **QFL** | $-|\hat{y}-\sigma|^\beta\log(\sigma_t)$ | 连续标签 + 质量感知 | 连续标签 | Dynamic FCOS |

## 7. 小目标检测中的分类损失

小目标 (32x32px 以下) 在 640x640 输入中，P3 特征图上仅占 1-2 个 cell，正负比可达 **1:600+**。

| 问题 | 具体表现 | 影响 |
|------|---------|------|
| 梯度主导 | 简单负样本梯度累积远超正样本 | 正样本信号被淹没 |
| 检测遗漏 | 小目标训练中贡献不足 | Recall 极低 |
| 特征退化 | 模型倾向预测"背景" | 前景特征学习不充分 |

**应对策略**：
1. **Focal Loss** ($\gamma=2, \alpha=0.25$)：降低简单负样本权重
2. **更高 $\alpha$ 值**：极端不平衡时提升至 0.5-0.75
3. **BCE + 动态采样**：如 YOLOv8 TaskAlignedAssigner 动态调整正负比
4. **QFL/VFL**：通过质量标签聚焦高质量预测

## 8. YOLOv8 中 BCE 的使用方式

YOLOv8 采用 **Decoupled Head**，分类和回归解耦：

```
Backbone → Neck (FPN+PAN) → Decoupled Head
                              ├─ Cls Head → Sigmoid → BCE Loss
                              └─ Reg Head → Distribution → DFL + CIoU Loss
```

分类损失源码（简化）：

```python
loss = (
    F.binary_cross_entropy_with_logits(
        pred_scores, target_scores, reduction='none'
    )
    * target_scores   # 仅对正样本类别计算
).sum() / norm_factor
```

**软标签机制**：TaskAlignedAssigner 生成质量感知的软标签：

```
target_score = IoU^α_cls × p^β_cls    (α_cls=1.0, β_cls=6.0)
```

IoU 高且分类置信度高的样本获得高标签值，本质融合了 VFL/QFL 的质量感知思想。

**为何不用 Focal Loss**：TaskAlignedAssigner 的动态分配已缓解不平衡；软标签已融入质量感知；BCE 梯度更稳定、超参更少。
